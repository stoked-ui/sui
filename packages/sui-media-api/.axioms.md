# AXIOMS

## AX-MOD-MEDIAAPI-001: Media-API scope boundary is binding
This module exposes media-component endpoints only. New business / domain endpoints (products, clients, licenses, invoices, billing, non-media users, marketing surfaces, etc.) MUST NOT be added under `packages/sui-media-api`; they belong in `docs/pages/api/*`. Allowed controller surfaces today are limited to `Health`, `auth`, `Media`, `Uploads`, and the Swagger UI mount.

### Applies To
- `packages/sui-media-api/src/app.module.ts`
- `packages/sui-media-api/src/**/*.controller.ts`
- `packages/sui-media-api/src/**/*.module.ts`

### Acceptance Checks
- manual: grep new/changed `*.controller.ts` files under `packages/sui-media-api/src` and confirm every route maps to media, uploads, auth (for the media surface), or health. Reject anything else.
- `rg -n "@Controller\(" packages/sui-media-api/src` — review the resulting list against the allowed surface (`auth`, `media`, `uploads`, `health`).

### Enforcement
computational

## AX-MOD-MEDIAAPI-002: HTTP / DTO contract is the client wire format
The route shape under `/auth`, `/media`, `/uploads`, the `API_PATH_PREFIX` defaults (`/v1` standalone, `/api` Lambda), and the DTO field names in `src/media/dto/*` and `src/uploads/dto/*` are consumed by `@stoked-ui/media` (`MediaApiClient`, `UploadClient`, Next.js proxy `createMediaHandler`) and by the exported OpenAPI artifact under `packages/sui-media-api/docs/`. Renames, removals, or shape changes are breaking changes that require a coordinated update of the browser client, the OpenAPI export, and any host-app SDKs in the same governed task.

### Applies To

- `packages/sui-media-api/src/uploads/dto/**`

## AX-MOD-MEDIAAPI-003: Multipart upload bounds are protocol-level invariants
The upload session protocol is fixed at: chunk size 5 MiB ≤ N ≤ 100 MiB (default 10 MiB / `UploadsService.DEFAULT_CHUNK_SIZE`), 1–50 presigned URLs per request (`MAX_URLS_PER_REQUEST`), `SESSION_EXPIRATION_DAYS = 7`, multipart presign lifetime 1 hour (`S3Service.MULTIPART_URL_EXPIRY`), and `InitiateUploadDto.contentType` matches `^(video|image)\/[a-z0-9.+-]+$`. The browser `UploadClient` enforces the matching bounds; loosening or tightening either side in isolation desynchronizes the protocol and breaks live uploads.

### Applies To
- `packages/sui-media-api/src/uploads/uploads.service.ts`
- `packages/sui-media-api/src/uploads/dto/**`
- `packages/sui-media-api/src/s3/s3.service.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/media-api test -- uploads`
- `pnpm --filter @stoked-ui/media-api test:e2e`
- manual: cross-check `chunkSize` / `partNumber` / MIME bounds against `packages/sui-media/src/api/UploadClient.ts` before merging.
- `grep -q "DEFAULT_CHUNK_SIZE = 10 \* 1024 \* 1024" packages/sui-media-api/src/uploads/uploads.service.ts && grep -q "MAX_URLS_PER_REQUEST = 50" packages/sui-media-api/src/uploads/uploads.service.ts && grep -q "SESSION_EXPIRATION_DAYS = 7" packages/sui-media-api/src/uploads/uploads.service.ts && grep -qF "^(video|image)\/[a-z0-9.+-]+$" packages/sui-media-api/src/uploads/dto/index.ts` — protocol constants and MIME regex unchanged.

### Enforcement

## AX-MOD-MEDIAAPI-004: JWT auth contract and dev-only `x-user-id` fallback are observable behavior
Production auth requires `Authorization: Bearer <jwt>`. In dev (`NODE_ENV !== 'production'`), the media-scoped `AuthGuard` (`src/media/guards/auth.guard.ts`) and `UploadsController` additionally accept `x-user-id` (header) or `?userId=` (query). The JWT secret resolution order — SST `Resource.JWT_SECRET` → dynamic `import('sst')` → `JWT_SECRET` env → `'dev-secret-change-me'` — and the JWT payload shape `{ sub, email, role, name }` are consumed by `AuthGuard`, the `@UserId()` decorator, and `JwtStrategy`. Changing any of these (including reordering the secret-resolution chain) silently invalidates issued tokens or breaks local tooling and the docs-site dev server at port 5199.

### Applies To
- `packages/sui-media-api/src/auth/auth.module.ts`
- `packages/sui-media-api/src/auth/auth.service.ts`
- `packages/sui-media-api/src/auth/strategies/jwt.strategy.ts`
- `packages/sui-media-api/src/auth/guards/**`
- `packages/sui-media-api/src/media/guards/auth.guard.ts`
- `packages/sui-media-api/src/media/decorators/current-user.decorator.ts`
- `packages/sui-media-api/src/uploads/uploads.controller.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/media-api test -- auth`
- `pnpm --filter @stoked-ui/media-api test:e2e`
- manual: start the docs site (`pnpm docs:dev`, port 5199) and confirm `useMediaList()` / upload flows work using `x-user-id` against a local dev server.
- `grep -q "dev-secret-change-me" packages/sui-media-api/src/auth/auth.module.ts` — the final JWT secret fallback in the resolution chain is preserved.

## AX-MOD-MEDIAAPI-005: Soft-delete is the default for `DELETE /media/:id`
`DELETE /media/:id` is a soft-delete unless the caller passes `?hardDelete=true`. `POST /media/:id/restore` recovers soft-deleted records. `GET /media/api/info` is the only `/media/*` route that is unauthenticated. Inverting the soft-delete default, removing the restore route, or locking down the public info route is a breaking change for the browser client (`MediaApiClient.deleteMedia` / `restoreMedia`) and docs-site surfaces that surface API metadata.

### Applies To
- `packages/sui-media-api/src/media/media.controller.ts`
- `packages/sui-media-api/src/media/media.service.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/media-api test -- media.service`
- `pnpm --filter @stoked-ui/media-api openapi:export` — confirm `DELETE /media/:id`, `POST /media/:id/restore`, and `GET /media/api/info` still appear and `GET /media/api/info` has no security requirement.

### Enforcement

## AX-MOD-MEDIAAPI-006: External integration contracts (S3, FFmpeg, Mongo, SST) must not change unilaterally
External contracts that the deployment surface depends on — system `ffmpeg`/`ffprobe` binaries (paths overridable via `FFMPEG_PATH`/`FFPROBE_PATH`), AWS S3 multipart commands + 1-hour presign lifetime via `@aws-sdk/client-s3` v3, MongoDB connection options (`appName: 'stoked-ui-media-api'`, 5 s connect/serverSelection timeouts, 3-retry policy), the four feature schemas registered from `@stoked-ui/common-api` (`VideoFeature`, `ImageFeature`, `FileFeature`, `UploadSessionFeature`), Lambda binary content-type allow-list, CORS allow-list semantics (`ALLOWED_ORIGINS` env + localhost-in-dev branch + production fallback list), and env-var names (`PORT`, `API_PATH_PREFIX`, `MONGODB_URI`, `JWT_SECRET`, `AWS_REGION`, `AUTH_AUTO_DOMAINS`, etc.) — must not change without a coordinated migration that updates `.env.example`, `Dockerfile`, `k8s/`, `deploy.sh`, SST config, and the OpenAPI export in the same governed task.
