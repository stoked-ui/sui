# AXIOMS

## AX-MOD-SUICOMMONAPI-001: Single Public Entry Point
All public exports of `@stoked-ui/common-api` must flow through `packages/sui-common-api/src/index.ts`. Adding, removing, or renaming an export — including any `*Feature` (`ModelDefinition`), schema class, DTO, or decorator — is a contract change that requires a governed task with downstream consumer review (`sui-media-api`, and any future API service that registers `MongooseModule.forFeature([...])` from this package).

### Applies To
- `packages/sui-common-api/src/index.ts`
- `packages/sui-common-api/src/{models,dtos,decorators}/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api typescript`
- `pnpm --filter @stoked-ui/common-api build`
- `pnpm --filter @stoked-ui/media-api typescript`
- manual: grep monorepo for deep imports from `@stoked-ui/common-api/...` before removing or renaming a re-export.

## AX-MOD-SUICOMMONAPI-002: Backend-Only Package — No Browser Imports
`@stoked-ui/common-api` depends on `@nestjs/common`, `@nestjs/mongoose`, `@nestjs/swagger`, `class-validator`, and `mongoose`. It must never be imported from a browser package or browser-bundled code (`sui-editor`, `sui-timeline`, `sui-file-explorer`, `sui-media`, `sui-docs`, `docs/` client components). Browser code that needs shared types must import from `@stoked-ui/common` instead.

### Applies To
- `packages/sui-common-api/src/**/*.ts`
- Any consumer importing `@stoked-ui/common-api`

### Acceptance Checks
- `! grep -RE "@stoked-ui/common-api" packages/sui-editor packages/sui-timeline packages/sui-file-explorer packages/sui-media packages/sui-docs docs/src docs/pages docs/components 2>/dev/null`
- manual: confirm the only in-tree consumers are server-side packages (`sui-media-api` and any future API/Lambda runtime).

## AX-MOD-SUICOMMONAPI-003: `swapId` and `DefaultSchemaOptions` Are Wire-Format Contract
The `swapId` transform (`_id -> id` string, recursive into nested objects/arrays, plus `delete __v`) and `DefaultSchemaOptions` (`timestamps: true`, `virtuals: true`, transform installed on both `toObject` and `toJSON`) are observable wire-format contract for every Mongoose-document JSON response served by any consumer API. Changing the transform shape, removing timestamps, disabling virtuals, or bypassing the transform on a response is a breaking change for every client (browser SDKs, admin UIs, embed surfaces).

### Applies To
- `packages/sui-common-api/src/decorators/defaultSchemaOptions.ts`
- `packages/sui-common-api/src/decorators/stdschema.decorator.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api build`
- `pnpm --filter @stoked-ui/media-api typescript`
- manual: hit a media-api list endpoint locally and confirm every document has a string `id`, never `_id` or `__v`.

## AX-MOD-SUICOMMONAPI-004: Mongoose Model Class Names Are Registered Names
Every `*Feature: ModelDefinition` exported by this package uses `<Class>.name` as its registered Mongoose model name. Renaming a model class (e.g., `Media -> MediaItem`) changes the registered name and breaks every `@InjectModel(<Class>.name)` injection and every `ref: "<Class>"` string across the monorepo. Renames require a coordinated migration touching all consumers (today: `sui-media-api`, future API services per `.stokd/projects/*/prd.md`).

### Applies To
- `packages/sui-common-api/src/models/*.model.ts` (every `@StdSchema`-decorated class)

### Acceptance Checks
- `grep -RE "@InjectModel\(|ref:\s*['\"]" packages docs --include='*.ts' --include='*.tsx' | grep -v node_modules | grep -v build`
- `pnpm --filter @stoked-ui/media-api typescript`

## AX-MOD-SUICOMMONAPI-005: `InitiateUploadDto` Validation Is Multi-Client Contract
The validation rules on `InitiateUploadDto` (and the `UploadPartStatus` / `UploadSessionStatus` unions on `UploadSession`) are the wire contract for every upload client — browser SDK, mobile, and any server-to-server uploader. The mime-type regex `^(video|image)\/[a-z0-9.+-]+$`, the `totalSize >= 1` minimum, the `chunkSize` bounds `5 MiB..100 MiB` (the lower bound is an S3 protocol requirement), and the part-status enum values must not change without a coordinated migration with all upload clients. The `progress`/`completedPartsCount`/`uploadedBytes` virtuals only count parts whose `status === "completed"` — clients depend on this exact semantics.

### Applies To
- `packages/sui-common-api/src/dtos/upload.dto.ts`
- `packages/sui-common-api/src/models/uploadSession.model.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api typescript`
- `pnpm --filter @stoked-ui/media-api typescript`
- manual: boot media-api locally, initiate -> upload one part -> complete a multipart upload, and confirm progress virtuals and presigned-URL responses still behave correctly.

## AX-MOD-SUICOMMONAPI-006: Indexes On Production Models Are Performance Contract
The compound, text, sparse, and TTL indexes declared on `MediaSchema` (8 indexes including `media_text_search` with weights `title:10/tags:5/description:1`), `UploadSessionSchema` (TTL on `expiresAt`, compound `(userId, status, expiresAt)`, dedup `(hash, status)`), `LicenseSchema` (5 indexes), and `BlogPostSchema` back the public media list/search surfaces, resumable upload UI, license lookup workflows, and blog admin surfaces. Removing or renaming an index is a performance regression in production (Mongoose only creates missing indexes — orphans require manual cleanup). Changing the TTL on `UploadSession.expiresAt` requires coordinating with the write-time default.

### Applies To
- `packages/sui-common-api/src/models/media.model.ts`
- `packages/sui-common-api/src/models/uploadSession.model.ts`
- `packages/sui-common-api/src/models/license.model.ts`
- `packages/sui-common-api/src/models/blogPost.model.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api build`
- manual: compare `db.<collection>.getIndexes()` in the target environment against the index declarations in source before merging an index change.

## AX-MOD-SUICOMMONAPI-007: `StdSchema` Decorator Deep Import Is The Compatibility Boundary
`StdSchema` registers schema metadata via a deep import: `@nestjs/mongoose/dist/storages/type-metadata.storage.js`. This path is the single most version-sensitive line in the package — bumping `@nestjs/mongoose` past v10 will likely require updating this import. Any version bump of `@nestjs/mongoose`, `@nestjs/swagger`, `@nestjs/common`, `mongoose`, or `class-validator` must be coordinated with `sui-media-api` and re-verified through a full schema-registration smoke (boot media-api, hit `/v1/api/docs`, perform a multipart upload).

### Applies To
- `packages/sui-common-api/src/decorators/stdschema.decorator.ts`
- `packages/sui-common-api/package.json` (`dependencies` block)

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api build`
- `pnpm --filter @stoked-ui/media-api typescript`
- manual: boot media-api locally and verify `MongooseModule.forFeature([...])` still resolves every registered feature.

## AX-MOD-SUICOMMONAPI-008: Existing Tests Must Pass Before Merge
All tests currently in `packages/sui-common-api` must pass before any change to this module is merged. The current state is documented in `.stokd/meta/packages/sui-common-api/SC_TEST.md` (no committed tests yet; planned suites cover decorators, models, and DTOs). Any test added here falls under the same rule — new failures block merge.

### Applies To

- `packages/sui-common-api/src/**/*.test.{ts,tsx}`
- `packages/sui-common-api/src/**/*.spec.{ts,tsx}`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common-api test`
- `pnpm --filter @stoked-ui/common-api typescript`
