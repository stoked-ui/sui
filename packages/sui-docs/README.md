# @stoked-ui/docs

Documentation building blocks for Stoked UI product docs, demos, code examples, markdown rendering, theme branding, and docs-site utilities.

This package is a shared React component library for documentation experiences. It is not the Next.js docs app itself.

## Installation

```bash
pnpm add @stoked-ui/docs react next @mui/material @mui/system @mui/icons-material @mui/base
```

Peer dependencies:

- `@mui/base`
- `@mui/icons-material`
- `@mui/material`
- `@mui/system`
- `next`
- `react`
- optional `@types/react`

Runtime dependencies include `react-runner`, `nprogress`, `clipboard-copy`, `fg-loadcss`, `clsx`, `prop-types`, and Babel runtime helpers.

## Quick Start

```tsx
import {
  BrandingCssVarsProvider,
  DocsProvider,
  MarkdownElement,
  HighlightedCode,
} from '@stoked-ui/docs';

const docsConfig = {
  LANGUAGES: ['en'],
  LANGUAGES_SSR: ['en'],
  LANGUAGES_IN_PROGRESS: [],
  LANGUAGES_IGNORE_PAGES: () => false,
};

export function DocsPage() {
  return (
    <BrandingCssVarsProvider>
      <DocsProvider config={docsConfig} defaultUserLanguage="en">
        <MarkdownElement renderedMarkdown="<h1>Editor</h1><p>Documentation content.</p>" />
        <HighlightedCode code={'const value = 1;'} language="tsx" />
      </DocsProvider>
    </BrandingCssVarsProvider>
  );
}
```

Render compiled rich markdown when the docs build has produced section metadata:

```tsx
import { RichMarkdownElement } from '@stoked-ui/docs';

<RichMarkdownElement demos={demos} renderedMarkdown={compiledSection} />;
```

## Primary Exports

- Providers: `BrandingCssVarsProvider`, `DocsProvider`, `CodeVariantProvider`, `CodeStylingProvider`, and i18n providers/hooks.
- Markdown and demo rendering: `MarkdownElement`, `RichMarkdownElement`, `Demo`, `DemoEditor`, `DemoSandbox`, `DemoToolbar`, `DemoToolbarRoot`, and `ReactRunner`.
- Code utilities: `HighlightedCode`, `HighlightedCodeWithTabs`, `CodeCopyButton`, `CodeSandbox`, and `StackBlitz`.
- Layout and UI: `InfoCard`, `Link`, `NProgressBar`, branding exports, and SVG icons.
- Utilities: `useLazyCSS`, `addHiddenInput`, `getFileExtension`, `getProductInfoFromUrl`, `stylingSolutionMapping`, translations, and docs component exports.
- Default export: `NProgressBar`.

## Integration Notes

- Wrap docs pages with `BrandingCssVarsProvider` before using components that rely on docs theme CSS variables.
- `DocsProvider` supplies docs configuration and language context to downstream components.
- `Demo`, `DemoToolbar`, and `ReactRunner` expect demo metadata generated by the docs build pipeline.
- The package publishes from `build/`; deep import paths are defined in `package.json` exports and generated/copied during the build.
- Next.js router integration is used by `Link` and `NProgressBar`.
