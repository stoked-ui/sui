# AXIOMS

## AX-MOD-SUIDOCS-001: Subpath exports and barrel must stay in sync with build:copy-files

Every entry in `package.json#exports` (e.g. `./Demo`, `./HighlightedCode`, `./MarkdownElement`, `./SandboxDependencies`, `./i18n`, `./branding`, `./svgIcons`, `./translations`, `./useLazyCSS`, `./addHiddenInput`, `./getFileExtension`, `./getProductInfoFromUrl`, `./stylingSolutionMapping`) must (a) be re-exported from `src/index.ts` or its sub-barrels (`src/components/index.ts`), and (b) have a matching pair in the `build:copy-files` script so the flat per-component directory is populated at publish time. Removing or renaming a subpath is a breaking change for the `docs/` site and any external consumer importing `@stoked-ui/docs/<Component>`.

### Applies To
- `packages/sui-docs/package.json` (`exports`, `scripts.build:copy-files`)
- `packages/sui-docs/src/index.ts`
- `packages/sui-docs/src/components/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/docs build` completes (executes legacy + node + stable + types + copy-files + postbuild)
- `manual: grep '"./<NewSubpath>"' packages/sui-docs/package.json` returns both an `exports` entry and a `build:copy-files` mapping
- `pnpm --filter stokedui-com typescript` (the docs app's `tsc -p tsconfig.json && tsc -p scripts/tsconfig.json`; there is no `docs:typecheck` script) succeeds after the change

### Enforcement
suite

## AX-MOD-SUIDOCS-002: BrandingCssVarsProvider cssVarPrefix must remain "muidocs"

`BrandingCssVarsProvider` hard-codes `cssVarPrefix: 'muidocs'`. Hundreds of MDX strings, CSS rules, and themed components across the `docs/` site reference `var(--muidocs-palette-*)` / `var(--muidocs-*)`. The prefix is contractual — renaming it silently breaks every themed surface (light/dark, palette, typography overrides). Any change requires a coordinated migration of every `--muidocs-*` reference across the monorepo.

### Applies To
- `packages/sui-docs/src/BrandingCssVarsProvider/BrandingCssVarsProvider.tsx`
- `packages/sui-docs/src/branding/brandingTheme.ts`

### Acceptance Checks
- `manual: grep -r "muidocs" packages/sui-docs/src docs/src docs/pages docs/styles | wc -l` is non-zero before any rename and zero (or fully migrated) after
- `manual: pnpm docs:dev` (port 5199) — verify light and dark mode render correctly on `/`, `/pricing`, and an MDX page
- `grep -n "cssVarPrefix: 'muidocs'" packages/sui-docs/src/BrandingCssVarsProvider/BrandingCssVarsProvider.tsx` returns exactly one hit (currently line 66)

### Enforcement
computational

## AX-MOD-SUIDOCS-003: DocsProvider config shape is a public contract

`DocsConfig` has exactly four fields — `LANGUAGES`, `LANGUAGES_SSR`, `LANGUAGES_IN_PROGRESS`, and `LANGUAGES_IGNORE_PAGES(pathname): boolean` (see `DocsProvider.tsx` lines 4–9; there is no `productIdentifier` on `DocsConfig` — that identifier lives in the docs app's own `_app.js` config). It is consumed by `Link`, `pathnameToLanguage`, and the `docs/` site's `_app`. `useDocsConfig` deliberately throws (`'Could not find docs config context value; please ensure the component is wrapped in a <DocsProvider>'`) outside `DocsProvider`. Adding required fields, removing fields, or changing the throw behavior is a breaking change for every consumer.

### Applies To
- `packages/sui-docs/src/DocsProvider/DocsProvider.tsx`

### Acceptance Checks
- Existing `useDocsConfig`-outside-provider behavior is exercised by the test plan in `.stokd/meta/packages/sui-docs/SC_TEST.md` §1.5; any change must keep that test green or update it under a governed task
- `manual: pnpm docs:dev` and visit a localized URL (`/zh/...` or `/fr/...`) — language-prefixed links must still resolve
- `node -e "const s=require('fs').readFileSync('packages/sui-docs/src/DocsProvider/DocsProvider.tsx','utf8'); const m=s.match(/interface DocsConfig \{([^}]*)\}/)[1]; const fields=(m.match(/^\s*[A-Z_]+/gm)||[]).length; process.exit(fields===4?0:1)"` — `DocsConfig` still declares exactly four fields

### Enforcement
computational

## AX-MOD-SUIDOCS-004: Code variant/styling persistence precedence is URL hash → cookie → default

`CodeVariantProvider` (default `'TS'`) and `CodeStylingProvider` (default `'SUI System'`) initialize from URL hash first, then cookie, then the hard-coded default. The mapping from `CODE_STYLING` → URL slug lives in `stylingSolutionMapping.ts`. Reordering this precedence flips the demo language/styling globally for returning visitors.

### Applies To
- `packages/sui-docs/src/components/codeVariant.tsx`
- `packages/sui-docs/src/components/codeStylingSolution.tsx`
- `packages/sui-docs/src/components/stylingSolutionMapping.ts`
- `packages/sui-docs/src/components/constants.js`

### Acceptance Checks
- `manual: pnpm docs:dev`, open a doc page with a `Demo` block and append `#demo.js` — the JS tab must be active on first paint; remove the hash and refresh — the previous cookie value must win; clear cookies — the default must apply
- `npx tsc --noEmit -p packages/sui-docs/tsconfig.json` (project-isolated type-check; `sui-docs` has no standalone `typescript` package script — verified exit 0, 0 errors on 2026-06-06)

### Enforcement
suite

## AX-MOD-SUIDOCS-005: SandboxDependencies must keep the window.muiDocConfig override surface

`SandboxDependencies` (in `src/components/Dependencies.ts`) reads `window.muiDocConfig` and consults `csbIncludePeerDependencies`, `csbGetVersions`, and `postProcessImport`. The `docs/` site uses these hooks for experimental dependency overrides. Removing, renaming, or changing the call signatures breaks sandbox launches in production. `@types/*` packages must continue to be injected for non-bundled-types imports.

### Applies To
- `packages/sui-docs/src/components/Dependencies.ts`
- `packages/sui-docs/src/components/CodeSandbox.tsx`
- `packages/sui-docs/src/components/StackBlitz.tsx`

- `npx tsc --noEmit -p packages/sui-docs/tsconfig.json` (project-isolated type-check; `sui-docs` has no standalone `typescript` package script — verified exit 0, 0 errors on 2026-06-06)
- `grep -nE "csbIncludePeerDependencies|csbGetVersions|postProcessImport|muiDocConfig" packages/sui-docs/src/components/Dependencies.ts` returns all four hook names

### Enforcement

## AX-MOD-SUIDOCS-006: i18n lookup semantics are baked into MDX content

`useTranslate` looks up keys via dot-paths (`pages.foo.bar`), warns on missing keys via `console.error` (suppressed when `ignoreWarning: true` or in CI), and returns `'…'` when the requested language bundle is missing entirely. English (`src/translations/translations.json`) is the fallback bundle and is copied to `build/translations/` and `node/translations/` by the `postbuild` script (`copyTranslations.sh`). These semantics are relied on by thousands of MDX strings and must not be changed without a coordinated migration of consumer pages and tests.

### Applies To
- `packages/sui-docs/src/i18n/i18n.tsx`
- `packages/sui-docs/src/translations/translations.json`
- `packages/sui-docs/copyTranslations.sh`

### Acceptance Checks
- `pnpm --filter @stoked-ui/docs build` (postbuild must populate `build/translations/translations.json` and `node/translations/translations.json`)
- `manual: pnpm docs:dev` — force a missing translation key in a dev build; `console.error` must fire and the fallback string must render

### Enforcement
suite

## AX-MOD-SUIDOCS-007: Module is Next.js-coupled; consumers must run inside a Next.js app

The package declares `next ^13.5.1 || ^14` as a peer dependency, and exactly five source files import `next/router` / `next/link` (verified by grep): `BrandingCssVarsProvider.tsx` and its `MarkdownLinks.js` sibling, `Link/Link.tsx`, `components/DemoToolbar.tsx`, and `components/CodeCopy.tsx`. (Note: `codeVariant.tsx`, `codeStylingSolution.tsx`, and `MarkdownElement.tsx` read `window.location.hash` directly and do **not** import `next/router` — do not add them here on the assumption that they do.) Removing the Next.js coupling, or introducing a non-Next runtime path without an explicit isomorphic wrapper, is a breaking change for the consuming `docs/` app and the `examples/stoked-ui-nextjs-pages-router*` templates that mirror these primitives.

### Applies To
- `packages/sui-docs/src/BrandingCssVarsProvider/BrandingCssVarsProvider.tsx`
- `packages/sui-docs/src/BrandingCssVarsProvider/MarkdownLinks.js`
- `packages/sui-docs/src/Link/Link.tsx`
- `packages/sui-docs/src/components/DemoToolbar.tsx`
- `packages/sui-docs/src/components/CodeCopy.tsx`

### Acceptance Checks
- `grep -rnE "from ['\"]next/" packages/sui-docs/src` — Next.js imports remain confined to the five files listed above (single- and double-quote forms both matched); any new file with a `next/*` import must be added to this axiom
- `npx tsc --noEmit -p packages/sui-docs/tsconfig.json` (project-isolated type-check; `sui-docs` has no standalone `typescript` package script — verified exit 0, 0 errors on 2026-06-06)
- `manual: pnpm docs:dev` (port 5199) renders `/` and an MDX doc page without `router`/SSR errors
- `test "$(grep -rlE "from ['\"]next/" packages/sui-docs/src | wc -l | tr -d ' ')" = "5"` — exactly five source files import `next/*`

### Enforcement
computational
