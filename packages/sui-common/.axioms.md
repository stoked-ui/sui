# AXIOMS

## AX-SUI-COMMON-001: CalendarBooking export contract is semver-bound
The CalendarBooking component exported from @stoked-ui/common must export
`CalendarBookingProps`, `BookingFormData`, `TimeSlot`, and `DayColumn` from its
types file. Any structural change to these exported types is a semver-breaking
change and requires a version bump before publishing.

### Applies To
- `packages/sui-common/src/CalendarBooking/CalendarBooking.types.ts`
- `packages/sui-common/src/CalendarBooking/index.tsx`

### Acceptance Checks
- `grep -n "TimeSlot\|DayColumn\|CalendarBookingProps\|BookingFormData" packages/sui-common/src/CalendarBooking/CalendarBooking.types.ts` shows all four exported
- `pnpm typescript` in packages/sui-common exits 0

### Enforcement
computational

## AX-SUI-CALENDARBOOKING-001: Duration drag handle lives in the grid, not the form
The CalendarBooking duration drag handle (`data-testid="duration-drag-handle-bottom"`) MUST
be rendered inside the week grid on the selected slot block — never inside the booking form
section. The component test suite in `src/CalendarBooking/__tests__/` MUST cover: slot
selection sets `data-selected`, duration drag snap math (30px → +15 min), drag handle is
inside week-grid and NOT inside booking-form, and form submit disabled until name/email/phone
filled. These behaviors are the canonical test contract.

### Applies To
- `packages/sui-common/src/CalendarBooking/CalendarBooking.tsx`
- `packages/sui-common/src/CalendarBooking/__tests__/CalendarBooking.test.tsx`

### Acceptance Checks
- `grep -n "duration-drag-handle-bottom" packages/sui-common/src/CalendarBooking/CalendarBooking.tsx` shows handle inside the grid slot block
- `pnpm test --testPathPattern="CalendarBooking"` in packages/sui-common exits 0 with tests for selection, drag snap, handle placement, and form validation

### Enforcement
suite

## AX-MOD-SUICOMMON-001: `src/index.tsx` Is The Sole Public Barrel
The single public entry point of `@stoked-ui/common` is `src/index.tsx`; every public
export (LocalDb, ProviderState, Types, Ids, FetchBackoff, MimeType, Colors, interfaces,
SocialLinks, UserMenu, CalendarBooking, GrokLoader, the resize hooks) MUST flow through it.
Adding, removing, or renaming a barrel export is a contract change requiring a governed task,
and consumers MUST import from `@stoked-ui/common`, never from deep paths
(`@stoked-ui/common/src/...`). Staged surface such as `src/Dialog/Dialog.types.ts` is NOT
part of the contract until it is wired into the barrel — wiring it in is itself a governed change.

### Applies To
- `packages/sui-common/src/index.tsx`

### Acceptance Checks
- `grep -nE "export \* from|export \{" packages/sui-common/src/index.tsx` enumerates the public surface
- `! grep -rnE "from ['\"]@stoked-ui/common/(src|build)/" packages docs` — no deep imports across workspace boundaries
- `pnpm --filter @stoked-ui/common typescript` exits 0

### Enforcement
computational

## AX-MOD-SUICOMMON-002: `src/interfaces/**` Stays Server-Free And React-Free
Files under `src/interfaces/` (`publicity.ts`, `embed-visibility.ts`, `upload-types.ts`) are
re-exported by the server-side API packages (`@stoked-ui/common-api`, `@stoked-ui/media-api`),
so they MUST remain pure TypeScript types/constants with no imports of NestJS, Mongoose, the
AWS SDK, React, or Node-only modules. Introducing any such import breaks the API packages' build.

### Applies To
- `packages/sui-common/src/interfaces/*.ts`

### Acceptance Checks
- `! grep -rnE "from ['\"](@nestjs|mongoose|@aws-sdk|react|react-dom|fs|path|next)" packages/sui-common/src/interfaces`
- `pnpm --filter @stoked-ui/common-api typescript` exits 0

### Enforcement
computational

## AX-MOD-SUICOMMON-003: CalendarBooking Booking API Is A Docs-Owned HTTP Contract
`CalendarBooking` reaches the backend only through `GET {apiBaseUrl}/api/calendar/availability?date=YYYY-MM-DD`
(returning `{ slots: string[] }`) and `POST {apiBaseUrl}/api/calendar/book`. These are
non-media business routes owned by the docs Next.js API surface (`docs/pages/api/calendar/**`)
per `AX-REPO-MEDIA-API-BOUNDARY`; the component MUST NOT hard-code a non-`apiBaseUrl` origin,
and any change to a route path or response shape MUST be coordinated with the docs handler.

### Applies To
- `packages/sui-common/src/CalendarBooking/CalendarBooking.tsx`

### Acceptance Checks
- `grep -nE "/api/calendar/(availability|book)" packages/sui-common/src/CalendarBooking/CalendarBooking.tsx` shows both routes prefixed by `apiBaseUrl`
- `! grep -nE "fetch\(['\"]https?://" packages/sui-common/src/CalendarBooking/CalendarBooking.tsx` — no hard-coded absolute origin

### Enforcement
computational

## AX-MOD-SUICOMMON-004: Foundation Layer Has Zero Internal `@stoked-ui/*` Dependencies
`@stoked-ui/common` is the root of the package dependency graph; it MUST NOT declare any other
`@stoked-ui/*` package in its `dependencies` or `peerDependencies`, so every Stoked UI package
can depend on it without creating a cycle.

### Applies To
- `packages/sui-common/package.json`

### Acceptance Checks
- `node -e "const d=require('./packages/sui-common/package.json'); const all={...d.dependencies,...d.peerDependencies}; if(Object.keys(all).some(k=>k.startsWith('@stoked-ui/'))) { console.error('internal dep found'); process.exit(1); }"` exits 0

### Enforcement
computational

## AX-MOD-SUICOMMON-005: Existing Tests Must Stay Green Before Any Mutation
The Jest suites in `packages/sui-common/src/**/__tests__/` (`SocialLinks`, `SocialLinkField`,
`platformRegistry`, `CalendarBooking`) are the regression contract for this module and MUST
pass before any change to it is merged. New behavior added to a covered surface requires a
red→green test in the same change.

### Applies To
- `packages/sui-common/src/**/__tests__/*.test.{ts,tsx}`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common test` exits 0

### Enforcement
suite
