# AXIOMS

## AX-MOD-SUICOMMON-001: Single Public Entry Point
All public exports of `@stoked-ui/common` must flow through `packages/sui-common/src/index.tsx`. Adding, removing, or renaming an export from this file is a contract change that requires a governed task with downstream consumer review (`sui-editor`, `sui-timeline`, `sui-file-explorer`, `sui-media`, `sui-common-api`, `sui-media-api`, `docs/`).

### Applies To
- `packages/sui-common/src/index.tsx`
- `packages/sui-common/src/**/index.ts`
- `packages/sui-common/src/**/index.tsx`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common typescript`
- `pnpm --filter @stoked-ui/common build`
- manual: grep monorepo for deep imports from `@stoked-ui/common/...` before removing or renaming a re-export.

## AX-MOD-SUICOMMON-002: Zero Internal Stoked UI Dependencies
`@stoked-ui/common` is the foundation package and must not import from any other `@stoked-ui/*` workspace package. It may only depend on its declared `dependencies` (`@tempfix/idb`, `framer-motion`) and `peerDependencies` (`react`, `@mui/*`, `@emotion/*`).

### Applies To
- `packages/sui-common/src/**/*.{ts,tsx}`
- `packages/sui-common/package.json`

### Acceptance Checks
- `! grep -R "from '@stoked-ui/" packages/sui-common/src`
- manual: review `packages/sui-common/package.json` to confirm no `@stoked-ui/*` entry in `dependencies` or `peerDependencies`.

## AX-MOD-SUICOMMON-003: Client-Safe Interfaces Stay Server-Agnostic
Files under `packages/sui-common/src/interfaces/` are re-exported by `@stoked-ui/common-api` and `@stoked-ui/media-api` (NestJS). They must contain only TypeScript types and plain constants — no React, NestJS, Node, or DOM imports.

### Applies To
- `packages/sui-common/src/interfaces/**/*.ts`

### Acceptance Checks
- `! grep -RE "from 'react'|@nestjs|from 'fs'|from 'path'|window\.|document\." packages/sui-common/src/interfaces`
- `pnpm --filter @stoked-ui/common-api typescript`
- `pnpm --filter @stoked-ui/media-api typescript`

## AX-MOD-SUICOMMON-004: Browser-Only Side Effects Must Be SSR-Guarded
`LocalDb`, `VideoDb`, `useResize`, `useResizeWindow`, and any other code touching `window`, `document`, `indexedDB`, or `navigator` must remain safe to import in the Next.js `docs/` site under SSR. Module-load time must not throw when these globals are missing.

### Applies To
- `packages/sui-common/src/LocalDb/**/*.{ts,tsx}`
- `packages/sui-common/src/useResize/**/*.{ts,tsx}`
- `packages/sui-common/src/useResizeWindow/**/*.{ts,tsx}`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common build`
- manual: `pnpm docs:dev` (port 5199) — server-render any page that transitively imports the module; verify no `window is not defined` / `indexedDB is not defined` errors at boot.

## AX-MOD-SUICOMMON-005: Global Prototype Extension `Array.prototype.mergeWith` Is Observable Contract
Importing `src/Types/mergeWith.ts` installs `Array.prototype.mergeWith` as a global side effect. Renaming, removing, or changing its signature `(other: T[], key: keyof T) => T[]` is a breaking change for every consumer that calls `arr.mergeWith(...)`.

### Applies To
- `packages/sui-common/src/Types/mergeWith.ts`
- `packages/sui-common/src/Types/Types.ts`

### Acceptance Checks
- `grep -RE "mergeWith\(" packages --include='*.ts' --include='*.tsx' | grep -v node_modules`
- `pnpm --filter @stoked-ui/common typescript`

## AX-MOD-SUICOMMON-006: Provider State / Settings Proxy Semantics
`createProviderState` and `createSettings` (`src/ProviderState/`) are depended on by `sui-editor` and `sui-timeline` providers. The dot-path get/set with autovivified intermediate objects on `createSettings`, and the `enable/disable/toggleFlags` + `addTriggers/removeTriggers` cascade on `createProviderState`, are observed behavior — including the documented `settings`-vs-`this.settings` quirk inside `checkTriggers`. Behavioral changes require coordinated updates with provider consumers.

### Applies To
- `packages/sui-common/src/ProviderState/ProviderState.ts`
- `packages/sui-common/src/ProviderState/Settings.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common test`
- `pnpm --filter @stoked-ui/editor typescript`
- `pnpm --filter @stoked-ui/timeline typescript`
- manual: open the editor demo at `localhost:5199` and verify project save/load + flag toggles in the timeline still work.

## AX-MOD-SUICOMMON-007: Existing Tests Must Pass Before Merge
All tests currently in `packages/sui-common` (Jest, jsdom) must pass before any change to this module is merged. Today this is the SocialLinks suite under `src/SocialLinks/__tests__/` — additions to the test surface fall under this same rule.

### Applies To
- `packages/sui-common/src/**/__tests__/**/*.{ts,tsx}`
- `packages/sui-common/src/**/*.test.{ts,tsx}`

### Acceptance Checks
- `pnpm --filter @stoked-ui/common test`
