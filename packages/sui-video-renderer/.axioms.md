# AXIOMS

## AX-MOD-VIDEO-RENDERER-001: `video-compositor` is the single composition engine for both WASM and CLI
The `video-compositor` rlib (`compositor/`) is the sole frame-composition engine; `wasm-preview` and `video-renderer-cli` are thin adapters that depend on it via `path = "../compositor"`. Composition logic (blending, transforms, effects, text, keyframes, timeline) MUST live in `video-compositor`, not be duplicated in an adapter. The re-exported surface in `compositor/src/lib.rs` (`Compositor`, `Layer`, `Transform`, `BlendMode`, `Effect`, `Frame`, `Timeline`, `TimelineLayer`, `AnimatedLayer`, the `keyframe`/`cache`/`text`/`types` exports, and `Error`/`Result`) is the contract both adapters compile against; a breaking change to it requires updating both adapters in the same governed task.

### Applies To
- `packages/sui-video-renderer/compositor/src/lib.rs`
- `packages/sui-video-renderer/compositor/src/**`

### Acceptance Checks
- `cd packages/sui-video-renderer && cargo test --workspace` (or `pnpm video-renderer:test`) passes.
- `cd packages/sui-video-renderer && cargo build --workspace` builds all three crates.
- manual: any new compositing capability is added to `video-compositor` and merely surfaced (not re-implemented) by `wasm-preview` / `cli`.

## AX-MOD-VIDEO-RENDERER-002: The `video-compositor` default build MUST stay `wasm32`-safe
`video-compositor` compiles to WebAssembly as the dependency of `wasm-preview`. The default feature set is empty; `rayon` and `tokio` live behind the `native` feature and `ffmpeg-next` behind `native-video`, and native-only items (`LayerContent::Video`, `Layer::video`, `compositor/src/video.rs`, parallel blending) are gated `#[cfg(not(target_arch = "wasm32"))]` / `#[cfg(feature = "rayon")]`. No code path reachable from the default build may import a non-WASM-safe crate (rayon, tokio, ffmpeg-next) or call `std::fs`/threads. The WASM build uses the sequential blend path.

### Applies To
- `packages/sui-video-renderer/compositor/Cargo.toml`
- `packages/sui-video-renderer/compositor/src/compositor.rs`
- `packages/sui-video-renderer/compositor/src/layer.rs`
- `packages/sui-video-renderer/compositor/src/video.rs`

### Acceptance Checks
- `cd packages/sui-video-renderer/wasm-preview && wasm-pack build --target web --out-dir ../pkg` succeeds (or `pnpm video-renderer:build-wasm`).
- `cd packages/sui-video-renderer && cargo build -p video-compositor` (default features) builds without rayon/tokio/ffmpeg.
- manual: new native-only code is gated behind `cfg(not(target_arch = "wasm32"))` or the `native`/`native-video` feature.

## AX-MOD-VIDEO-RENDERER-003: The `WasmLayer` JSON shape and string enums are the editor⇄WASM contract
`PreviewRenderer.render_frame` accepts a JSON string deserialized into `Vec<WasmLayer>` (`wasm-preview/src/lib.rs`). The `WasmLayer` field set (`id`, `type`, `color`, `video_element_id`, `image_url`, `transform{x,y,scale_x,scale_y,rotation,opacity}`, `blend_mode`, `visible`, `z_index`), the `type` strings (`"solidColor" | "image" | "video" | "text"`), and the `blend_mode` strings mapped in `convert_layer` are a runtime (untyped) contract with `@stoked-ui/editor`'s `actionMapper.ts`. Changing any field name, layer-type string, or blend-mode string MUST be mirrored in `packages/sui-editor/src/WasmPreview/{actionMapper.ts,types.ts}` and the typings in `wasm-module.d.ts` in the same change — a mismatch silently drops or mis-renders layers (no compile error).

### Applies To
- `packages/sui-video-renderer/wasm-preview/src/lib.rs`
- `packages/sui-editor/src/WasmPreview/actionMapper.ts`
- `packages/sui-editor/src/WasmPreview/wasm-module.d.ts`

### Acceptance Checks
- `cd packages/sui-video-renderer && cargo test -p wasm-preview` passes.
- `pnpm --filter @stoked-ui/editor typescript` passes after any shape change.
- manual: grep `packages/sui-editor/src/WasmPreview` for each changed `WasmLayer` field / layer-type / blend-mode string and confirm both sides agree.

## AX-MOD-VIDEO-RENDERER-004: `PreviewRenderer` exported method surface is the JS contract
The `#[wasm_bindgen]` surface of `wasm-preview` (`init`/default `__wbg_init`, `PreviewRenderer::new(canvas, width, height)`, `render_frame`, `render_frame_at_time`, `clear`, `get_metrics`, `cache_image`, `clear_image_cache`, `is_image_cached`, the `Color` class, `benchmark_composition`) is the JavaScript-facing API the editor binds to. Removing, renaming, or changing the arity of any of these is a breaking change for `@stoked-ui/editor`; the hand-written ambient typings in `packages/sui-editor/src/WasmPreview/wasm-module.d.ts` and the `PreviewRendererInstance` interfaces in `EditorEngine.ts` / `CompositorController.ts` / `useWasmRenderer.ts` MUST be updated to match.

### Applies To
- `packages/sui-video-renderer/wasm-preview/src/lib.rs`
- `packages/sui-editor/src/WasmPreview/wasm-module.d.ts`
- `packages/sui-editor/src/EditorEngine/EditorEngine.ts`

### Acceptance Checks
- `cd packages/sui-video-renderer/wasm-preview && wasm-pack build --target web --out-dir ../pkg` regenerates `pkg/wasm_preview.d.ts` with the changed surface.
- `pnpm --filter @stoked-ui/editor typescript` passes.
- manual: every renamed/removed method is reflected in `wasm-module.d.ts` and the editor `PreviewRendererInstance` interfaces.

## AX-MOD-VIDEO-RENDERER-005: The editor consumes the WASM artifact from `pkg/` via the `@stoked-ui/video-renderer-wasm` alias
The build product consumed by `@stoked-ui/editor` is the `wasm-pack` output in `packages/sui-video-renderer/pkg/`, resolved through the `docs/next.config.mjs` webpack alias `@stoked-ui/video-renderer-wasm` → `packages/sui-video-renderer/pkg` with `experiments.asyncWebAssembly: true`. The canonical build is `pnpm build:wasm` (`scripts/build-wasm.sh`, target `web`), which also rewrites `pkg/package.json` `name` to `@stoked-ui/video-renderer-wasm`. The output directory MUST remain `pkg/` (not `wasm-preview/pkg/`), and any change to the output path, artifact filenames, or package name MUST update `scripts/build-wasm.sh`, the root `package.json` wasm scripts, and the `next.config.mjs` alias together.

### Applies To
- `packages/sui-video-renderer/scripts/build-wasm.sh`
- `packages/sui-video-renderer/pkg/**`
- `docs/next.config.mjs`

### Acceptance Checks
- `pnpm build:wasm` produces `packages/sui-video-renderer/pkg/wasm_preview_bg.wasm` and a `pkg/package.json` named `@stoked-ui/video-renderer-wasm`.
- `grep -n "video-renderer-wasm" docs/next.config.mjs` shows the alias resolving to `packages/sui-video-renderer/pkg` and `asyncWebAssembly: true` present.
- manual: editor preview/render loads the WASM module without a missing-module error in the browser console (`AX-PROD-SUI-004` / `AX-REPO-WASM-RENDERER-DEP`).

## AX-MOD-VIDEO-RENDERER-006: The `video-render` CLI contract — subcommands, `.sue` model, and FFmpeg dependency
The `video-render` binary (`cli/src/main.rs`) exposes exactly two subcommands, `render` and `info`, over the `.sue` JSON project model in `cli/src/project.rs` (`Project`/`ProjectTrack`/`TrackType`/`TrackAction`/`TrackTransform`). `parse_resolution` enforces the `WIDTHxHEIGHT` form and rejects zero dimensions; encoding and audio mux shell out to the **FFmpeg binary on `PATH`**. Renaming a subcommand/flag, changing the `.sue` field set or `TrackType` strings (`video|image|audio|text|solidcolor`), or removing the FFmpeg dependency is an external contract change requiring a coordinated update of the CLI guide (`packages/sui-video-renderer/docs/cli-guide.md`), the test fixtures (`cli/tests/fixtures/*.sue`), and any editor-side `.sue` producer.

### Applies To
- `packages/sui-video-renderer/cli/src/main.rs`
- `packages/sui-video-renderer/cli/src/project.rs`
- `packages/sui-video-renderer/cli/tests/**`

### Acceptance Checks
- `cd packages/sui-video-renderer && cargo test -p video-renderer-cli` passes (FFmpeg-dependent e2e cases self-skip when `ffmpeg -version` is unavailable).
- `cd packages/sui-video-renderer && cargo run -p video-renderer-cli -- info --input cli/tests/fixtures/simple.sue` parses and prints project info.
- manual: any `.sue` schema or CLI-flag change updates `docs/cli-guide.md` and the `cli/tests/fixtures/*.sue` files.

## AX-MOD-VIDEO-RENDERER-007: Compositor invariants — validated dimensions, z-order, and alpha compositing
`Compositor::new` MUST reject zero width or height with `Error::InvalidDimensions`. `compose` MUST render only `visible` layers, sorted ascending by `z_index` (bottom→top), applying opacity then transform then blend-mode compositing, and MUST produce a `Frame` exactly `width × height` regardless of layer transforms. The tiled (`compose_tiled`) and resolution-scaled (`compose_with_resolution_scale`) paths MUST produce output equivalent to `compose` for the same inputs (tiled is pixel-identical; scaled differs only by resampling). These behaviors are covered by the inline `compositor.rs` tests and are observable contracts.

### Applies To
- `packages/sui-video-renderer/compositor/src/compositor.rs`

### Acceptance Checks
- `cd packages/sui-video-renderer && cargo test -p video-compositor compositor::` passes (includes `test_invalid_dimensions`, `test_tiled_vs_standard_composition`, `test_compose_with_*_resolution_scale`).
- manual: a new render path keeps the `width × height` output-size guarantee and the z-index ordering.
