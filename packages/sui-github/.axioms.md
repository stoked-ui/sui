# AXIOMS

## AX-MOD-GITHUB-001: Public barrel is the contract
The exports from `src/index.ts` (components `GithubBranch`, `GithubCalendar`, `GithubCommit`, `GithubEvents`; handler factories `createGithubBranchHandler`, `createGithubCommitHandler`, `createGithubContributionsHandler`, `createGithubEventsHandler`; server helpers `getBranchCompareDetails`, `getCommitDetails`, `getGithubContributions`, `getGithubEvents`, `getPullRequestDetails`, `githubEventsQuery`; and types `EventsQuery`, `GithubBranchData`, `GithubCommitData`, `GithubFileHighlight`) form the package's public contract. Renaming, removing, or changing the signatures of any of these requires a governed task with consumer review (docs site demos under `docs/data/github/**` and `docs/pages/api/github/**`).

### Applies To
- `packages/sui-github/src/index.ts`
- `packages/sui-github/src/apiHandlers/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/github typescript`
- `pnpm --filter @stoked-ui/github build`
- manual: grep `docs/` for each renamed export and update demos and API routes

## AX-MOD-GITHUB-002: Component ↔ handler query-string contract is symmetric
The query-string shape consumed by each component in `apiUrl` mode and produced by the corresponding handler factory must stay aligned. Specifically: contributions handlers exchange `username`, `from`, `to`; events handlers exchange `username`, `page`, `per_page`, `repo`, `action`, `date`, `description`; commit handlers exchange `owner`, `repo`, `ref`; branch handlers exchange `owner`, `repo`, `base`, `head`. Changing a param on one side without the other breaks any host app using the proxy seam.

### Applies To
- `packages/sui-github/src/apiHandlers/create*Handler.ts`
- `packages/sui-github/src/GithubCalendar/GithubCalendar.tsx`
- `packages/sui-github/src/GithubEvents/GithubEvents.tsx`
- `packages/sui-github/src/GithubCommit/GithubCommit.tsx`
- `packages/sui-github/src/GithubBranch/GithubBranch.tsx`
- `packages/sui-github/src/components/fetchGithubViewData.ts`

### Acceptance Checks
- manual: diff handler request parsing against component fetch URLs for the affected endpoint
- manual: smoke the matching demo page under `docs/data/github/docs/**` against a running docs site (port 5199)

## AX-MOD-GITHUB-003: Three render modes must produce identical output
`GithubCommit` and `GithubBranch` each support three modes — direct GitHub fetch, `apiUrl` proxy fetch, and snapshot `data` prop. All three modes must hydrate the same `GithubCommitData` / `GithubBranchData` shape and render the same view. Normalizers in `src/apiHandlers/githubApi.ts` (`normalizeGithubCommit`, `normalizeGithubFile`, `buildGithubContributors`, `summarizeGithubStats`, `parseGithubDiff`) are the single source of truth for that shape.

### Applies To
- `packages/sui-github/src/GithubCommit/**`
- `packages/sui-github/src/GithubBranch/**`
- `packages/sui-github/src/apiHandlers/githubApi.ts`
- `packages/sui-github/src/apiHandlers/getCommitDetails.ts`
- `packages/sui-github/src/apiHandlers/getBranchCompareDetails.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/github typescript`
- manual: visually compare a commit/branch via direct, proxied, and snapshot data sources

## AX-MOD-GITHUB-004: localStorage cache schema is versioned in-key
The events cache key `github_events_<username>` stores a `CachedData` payload with no embedded schema version. Any incompatible change to `CachedData` (in `src/types/github.ts`) must be paired with a key bump or explicit migration in `src/GithubEvents/GithubEvents.tsx`, otherwise stale browsers will fail to parse and silently fall back to an unauthenticated cold reload (which can hit the 60 req/h anon rate limit immediately).

### Applies To
- `packages/sui-github/src/GithubEvents/GithubEvents.tsx`
- `packages/sui-github/src/types/github.ts`
- `packages/sui-github/src/apiHandlers/getGithubEvents.ts`

### Acceptance Checks
- manual: clear `localStorage.github_events_*` and reload an events demo to verify cold-load path
- manual: load with a stale cache populated against the previous schema and confirm graceful invalidation

## AX-MOD-GITHUB-005: New GitHub event types require renderer + dispatcher updates
Adding support for a new GitHub event type requires three coordinated changes: (1) a new renderer under `src/GithubEvents/EventTypes/`, (2) a `case` arm in `processEvents` / event dispatcher within `src/GithubEvents/GithubEvents.tsx`, and (3) any new filter-facet wiring so client `filterEvents` and server `githubEventsQuery` stay in lockstep. Missing any of the three leaves events visible in raw payloads but invisible (or misclassified) in the UI.

### Applies To
- `packages/sui-github/src/GithubEvents/EventTypes/**`
- `packages/sui-github/src/GithubEvents/GithubEvents.tsx`
- `packages/sui-github/src/apiHandlers/getGithubEvents.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/github typescript`
- manual: feed a fixture payload containing the new event type and confirm it renders + filters correctly

## AX-MOD-GITHUB-006: Token-bearing code is server-only
`GITHUB_TOKEN` is read only inside `src/apiHandlers/**` (handler factories and the server helpers they invoke). React components in `src/GithubCalendar`, `src/GithubEvents`, `src/GithubCommit`, `src/GithubBranch` must never read `process.env.GITHUB_TOKEN` directly and must reach private data exclusively through the `apiUrl` proxy seam. Browser fallbacks (e.g. `github-contributions-api.jogruber.de`, unauthenticated `api.github.com`) are allowed but treated as best-effort.

### Applies To
- `packages/sui-github/src/GithubCalendar/**`
- `packages/sui-github/src/GithubEvents/**`
- `packages/sui-github/src/GithubCommit/**`
- `packages/sui-github/src/GithubBranch/**`
- `packages/sui-github/src/apiHandlers/**`

### Acceptance Checks
- manual: `grep -RE "GITHUB_TOKEN" packages/sui-github/src` — every hit must be inside `apiHandlers/`
- `pnpm --filter @stoked-ui/github build`

## AX-MOD-GITHUB-007: SSR-unsafe deps stay behind `next/dynamic`
`react-json-view` (used by event detail rendering) is not SSR-safe and must remain loaded via `next/dynamic` with `ssr: false`. Removing the dynamic guard causes hydration errors in the docs site.

### Applies To
- `packages/sui-github/src/GithubEvents/**`

### Acceptance Checks
- manual: `grep -RE "react-json-view" packages/sui-github/src` — every import path must go through a `next/dynamic({ ssr: false })` wrapper
- manual: load an events demo page from the docs site (port 5199) and confirm no hydration warnings
