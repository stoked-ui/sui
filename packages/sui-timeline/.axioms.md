# AXIOMS

## AX-MOD-TIMELINE-001: Engine Requires At Least One Controller At Construction
The `Engine` constructor must throw when instantiated without a non-empty `controllers` map. Code paths that instantiate `Engine` (directly or via subclass) must supply at least one `IController` keyed by its `id`.

### Applies To
- `packages/sui-timeline/src/Engine/Engine.ts`
- `packages/sui-timeline/src/Engine/Engine.types.ts`
- `packages/sui-timeline/src/TimelineProvider/TimelineProvider.tsx`

### Rationale
The engine's action lifecycle (`_dealEnter`, `_dealLeave`, `tickAction`) dispatches into `engine.controllers[action.controller]`. With no controllers registered, every action lookup silently no-ops and the playback loop appears to work while producing no media output. The constructor's `'No controllers set!'` guard exists to make that misconfiguration loud at boot rather than at first play.

### Acceptance Checks
- manual: instantiating `new Engine({})` (no `controllers` key) throws an error containing `No controllers set`.
- `grep -n "No controllers set" packages/sui-timeline/src/Engine/Engine.ts` returns a match.

### Enforcement
computational

## AX-MOD-TIMELINE-002: Public Exports In `src/index.ts` Are The Contract
Every symbol re-exported from `packages/sui-timeline/src/index.ts` is part of the package's public surface and is consumed by `@stoked-ui/editor` and the docs site. Removing or renaming an export — including default exports (`Timeline`, `TimelineFile`, `Engine`, `Controller`, etc.) and named re-exports from `./Engine`, `./TimelineProvider`, `./TimelineFile`, `./Controller`, `./Timeline*`, `./utils`, `./interface/const` — is a breaking change and requires a governed task with explicit downstream review.

### Applies To
- `packages/sui-timeline/src/index.ts`
- any module under `packages/sui-timeline/src/**` whose symbols are re-exported by `index.ts`

### Rationale
`EditorEngine extends Engine`, `EditorProvider` wraps `TimelineProvider`, `EditorFile extends TimelineFile`, and `EditorState extends TimelineState`. Silent removals appear as build-time TypeScript breaks across the monorepo, and renamings can land in docs MDX as undefined runtime imports.

### Acceptance Checks
- `pnpm --filter @stoked-ui/timeline typescript` exits 0.
- `pnpm --filter @stoked-ui/editor typescript` exits 0 (validates the editor still resolves every re-export it consumes).
- manual: if a top-level export is removed, the package version in `packages/sui-timeline/package.json` is bumped and the change is documented in the corresponding task.

### Enforcement
suite

## AX-MOD-TIMELINE-003: `utils/deal_data.ts` Is The Single Source Of Truth For Pixel/Time Math
All time-to-pixel and pixel-to-time conversions inside the timeline package (and any layout code that needs to translate between ruler time and screen position) must go through the exports of `packages/sui-timeline/src/utils/deal_data.ts` — `parserTimeToPixel`, `parserPixelToTime`, `parserTimeToTransform`, `parserTransformToTime`, `getScaleCountByRows`, `getScaleCountByPixel`, `parserActionsToPositions`. Do not inline equivalent arithmetic in components, controllers, or hooks.

### Applies To
- `packages/sui-timeline/src/Timeline/**`
- `packages/sui-timeline/src/TimelineCursor/**`
- `packages/sui-timeline/src/TimelineTime/**`
- `packages/sui-timeline/src/TimelineTrack/**`
- `packages/sui-timeline/src/TimelineTrackArea/**`
- `packages/sui-timeline/src/TimelineAction/**`
- `packages/sui-timeline/src/TimelinePlayer/**`

### Rationale
The ruler, cursor, action blocks, scroll resizer, and drag-line overlays all share the same `{startLeft, scale, scaleWidth}` parameter set. Duplicating the formula in a single component causes silent off-by-`startLeft` drift between the playhead and the action it is supposed to be over — bugs that pass type-check, pass per-component visual tests, and only show up when the user drags an action.

### Acceptance Checks
- `grep -rn "scaleWidth \* " packages/sui-timeline/src --include='*.ts' --include='*.tsx'` finds only matches inside `utils/deal_data.ts` and approved call sites that delegate to it. As of the 2026-06-06 refresh the only approved match outside `deal_data.ts` is `TimelineProvider/TimelineProviderFunctions.ts:122` (`fitScaleData`), which derives a new `scaleWidth` *setting* for zoom-to-fit rather than a per-element screen position; any additional match is a violation.
- manual: any new pixel/time math added to a component is implemented as a new helper inside `utils/deal_data.ts` (and exported from `src/utils/index.ts`) before being consumed.

### Enforcement
computational

## AX-MOD-TIMELINE-004: Existing Type-Check Spec Must Continue To Pass
The package's only checked-in spec (`packages/sui-timeline/src/themeAugmentation/themeAugmentation.spec.ts`) validates the MUI theme augmentation contract for `MuiTimeline*` slots. It must continue to compile and pass before any mutation to this module is merged. New tests added by future tasks must not break this spec.

### Applies To
- `packages/sui-timeline/src/themeAugmentation/**`
- `packages/sui-timeline/src/Timeline/**` (slot/class names referenced by augmentation)
- `packages/sui-timeline/src/TimelineLabels/**`, `TimelineTrack/**`, `TimelineCursor/**`, `TimelineTime/**`, `TimelineControls/**`, `TimelinePlayer/**`, `TimelineScrollResizer/**`

### Rationale
Renaming an MUI slot or class without updating `themeAugmentation/components.d.ts` is the most common silent break in this package, because the runtime keeps working but consumer theme overrides stop applying. The spec is the only automated guard.

### Acceptance Checks
- `pnpm --filter @stoked-ui/timeline typescript` exits 0.
- manual: if a `timeline*Classes.ts` slot name is added, removed, or renamed, the corresponding `Mui*` entry in `themeAugmentation/components.d.ts` is updated in the same change.

### Enforcement
suite

## AX-MOD-TIMELINE-005: Controller `id` Strings Are A Stable Lookup Key
A controller's `id` field (e.g. `'audio'` for `AudioController`) is the lookup key used by both `engine.controllers[action.controller]` and `TimelineFile`'s serialized `data` getter (which writes `controllerName` instead of an object reference). Renaming a controller's `id` is therefore an on-disk format change for every persisted `.sut` file that references it.

### Applies To
- `packages/sui-timeline/src/Controller/**`
- `packages/sui-timeline/src/TimelineFile/TimelineFile.ts` (`data` getter, `Controllers` static map, `actionInitializer`)
- `packages/sui-timeline/src/TimelineProvider/TimelineProvider.tsx` (controller registration)

### Rationale
`TimelineFile` round-trips through IndexedDB (`LocalDb` from `@stoked-ui/common`) and File System Access saves; the serialized form holds `controllerName` strings, not object references. Renaming a controller `id` without a migration leaves every existing project file pointing at a missing controller, and every track/action with that `controller` reference becomes invisible to the engine.

### Acceptance Checks
- manual: changing a controller's `id` value is accompanied by a versioned IDB migration in `getDbProps` and a backfill helper that rewrites `controllerName` in loaded files.
- `grep -rn "controllerName" packages/sui-timeline/src --include='*.ts' --include='*.tsx'` is reviewed to confirm every read site tolerates legacy values during the migration window.

### Enforcement
computational

## AX-MOD-TIMELINE-006: Engine Event Names And Veto Semantics Are Observable Behavior
The `EventTypes` union in `packages/sui-timeline/src/Engine/events.ts` defines the public emitter contract. The `before*` events (`beforeSetTime`, `beforeSetPlayRate`) honor a `false` return from any subscriber as a veto, in which case the corresponding `setTime` / `setPlayRate` call returns `false` and does not mutate state. Renaming an event, removing a `before*` veto path, or changing the payload shape is observable to every consumer that subscribes to the engine (notably `@stoked-ui/editor` and the docs showcase wrappers).

### Applies To
- `packages/sui-timeline/src/Engine/events.ts`
- `packages/sui-timeline/src/Engine/emitter.ts`
- `packages/sui-timeline/src/Engine/Engine.ts` (every `this.trigger(...)` and `this.bind(...)` site)

### Rationale
The editor's controllers (`Video`, `Image`, `Web`, `Compositor`) subscribe to engine events to drive seeking, preloading, and frame composition. A renamed event becomes a dead subscription with no compile-time warning. Removing a veto path breaks editor-level guards (e.g. "don't seek past loaded range") that rely on returning `false` from `beforeSetTime`.

### Acceptance Checks
- manual: any change to `EventTypes` is mirrored in `@stoked-ui/editor` subscribers and the docs showcase wrappers in the same task.
- `pnpm --filter @stoked-ui/editor typescript` exits 0 after the change.

### Enforcement
suite

## AX-MOD-TIMELINE-007: The `.sut` / `.sua` Mime Registrations Are A Persistence Contract
`StokedUiTimelineApp` must continue to register the `timeline-project` mime type with extension `.sut` (default input) and the `timeline-project-audio` mime type with extension `.sua` (default output); changing a subtype string or extension is an on-disk / IndexedDB format change requiring a coordinated migration.

### Applies To
- `packages/sui-timeline/src/Timeline/StokedUiTimelineApp.ts`
- `packages/sui-timeline/src/TimelineProvider/TimelineProvider.types.ts` (`getDbProps`)

### Rationale
`getDbProps(state.app.defaultInputFileType, ...)` derives the `LocalDb` (IndexedDB) database/store identity from the registered input mime type, and File System Access saves stamp the `.sut` / `.sua` extensions onto users' project files on disk. Renaming either the subtype or the extension silently orphans every previously persisted project — the app boots clean with no error, the user's work is simply "gone".

### Acceptance Checks
- `grep -n "'.sut'" packages/sui-timeline/src/Timeline/StokedUiTimelineApp.ts` and `grep -n "'.sua'" packages/sui-timeline/src/Timeline/StokedUiTimelineApp.ts` each return a match.
- `grep -n "timeline-project" packages/sui-timeline/src/Timeline/StokedUiTimelineApp.ts` returns matches for both `timeline-project` and `timeline-project-audio`.
- manual: any intentional change to these identifiers ships with a versioned IDB migration in `getDbProps` and a file-extension alias for loading legacy `.sut`/`.sua` files.

### Enforcement
computational
