# AXIOMS

## AX-MOD-SUI-EDITOR-001: Public exports in `src/index.ts` are the package contract
`packages/sui-editor/src/index.ts` is the sole publishable surface of `@stoked-ui/editor`. Removing, renaming, or narrowing any symbol re-exported from it (`Editor`, `EditorProvider`, `EditorEngine`, `EditorView`, `EditorFile`, `Controllers`, `createEditorFile`, `EditorVideoExampleProps`, `EditorExample`, plus the `export *` barrels for `EditorFile`, `EditorProvider`, `EditorView`, `Editor`, `EditorControls`, `models`, `hooks`, `Controllers`, `EditorAction`, `EditorTrack`) is a breaking change and requires a governed task with explicit consumer review (docs MDX demos under `docs/data/editor/**` and showcase components under `docs/src/**Editor*` consume these by name).

### Applies To
- `packages/sui-editor/src/index.ts`
- `packages/sui-editor/src/**/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/editor typescript`
- `pnpm --filter @stoked-ui/editor build:types`
- manual: grep `docs/data/editor` and `docs/src` for the changed export name and confirm every call site still compiles.

## AX-MOD-SUI-EDITOR-002: `EditorEngine` must remain a subclass of timeline `Engine` and route through the `Controllers` map
`EditorEngine` extends `Engine` from `@stoked-ui/timeline` and dispatches `draw`/`preload`/`enter`/`leave`/`update` per-frame via the `Controllers` record at `src/Controllers/Controllers.ts`. The engine class, the `IController` contract, and the action `type` → controller-key lookup must stay in sync. Adding an action type without registering a controller — or removing/renaming a controller without updating both `Controllers.ts` and `TimelineFile.Controllers` wiring in `EditorProvider` — silently drops frames for that track type.

### Applies To
- `packages/sui-editor/src/EditorEngine/**`
- `packages/sui-editor/src/Controllers/**`
- `packages/sui-editor/src/EditorProvider/EditorProvider.tsx`
- `packages/sui-editor/src/EditorAction/EditorAction.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/editor typescript`
- manual: load `/products/editor/` on the docs site (port 5199) with the `EditorExample` project and verify each action type (`audio`, `video`, `image`, `compositor`) renders frames with no controller-not-found warnings in the console.

## AX-MOD-SUI-EDITOR-003: WASM compositor is optional and must degrade to the canvas pipeline
`@stoked-ui/video-renderer-wasm` is declared as an `optionalDependency` resolving to `../sui-video-renderer/pkg`. `EditorEngine` enables the WASM path only when `_useWasm` is flipped and dynamically `import()`s the module; if the artifact is missing or the import fails, the canvas 2D pipeline (`_renderCtx`, `_renderDetailCtx`, controller `draw` calls) must continue to work. Changes to `WasmPreview/useWasmRenderer.ts`, `WasmPreview/actionMapper.ts`, `WasmPreview/types.ts`, or `CompositorController.ts` must preserve this dual-path behavior; both paths must remain runnable without modifying user code.

### Applies To
- `packages/sui-editor/src/WasmPreview/**`
- `packages/sui-editor/src/Controllers/CompositorController.ts`
- `packages/sui-editor/src/EditorEngine/EditorEngine.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/editor typescript`
- manual: with the WASM artifact present at `packages/sui-video-renderer/pkg` (the alias target in `docs/next.config.mjs`; the sibling `wasm-preview/pkg` is a stale duplicate), toggle `_useWasm` in `WasmPreviewDemo` and confirm rendering succeeds; with the artifact removed (or `_useWasm` left off), confirm the canvas path still renders `EditorExample` on the docs showcase.

## AX-MOD-SUI-EDITOR-004: IndexedDB schema for the `editor` database is versioned and forward-compatible
`src/db/init.ts` opens the `editor` IndexedDB database via `@tempfix/idb`. `EditorFile.updateStore()` persists project state into that store using the on-disk shape of `IEditorFileData` / `IEditorTrackData` / `IEditorFileAction`. Any change to that on-disk shape must ship with a versioned IDB upgrade path (bumping the DB version and supplying an `upgrade` migration); silently changing field names or types will brick existing users' saved projects and is forbidden.

### Applies To
- `packages/sui-editor/src/db/**`
- `packages/sui-editor/src/EditorFile/EditorFile.ts`
- `packages/sui-editor/src/EditorTrack/EditorTrack.ts`
- `packages/sui-editor/src/EditorAction/EditorAction.ts`

### Acceptance Checks
- manual: open a project saved against the previous schema in the docs editor (port 5199), reload, and confirm tracks, actions, and screenshots round-trip without console errors.
- manual: review the diff for a corresponding version bump and `upgrade` callback in `src/db/init.ts` whenever any persisted field on `EditorFile` / `EditorTrack` / `EditorAction` is renamed, removed, or retyped.

## AX-MOD-SUI-EDITOR-005: Tests in `packages/sui-editor` must continue to pass (and the missing harness must not be silently bypassed)
Existing test files under `packages/sui-editor/test/**` and any colocated `*.test.tsx` under `src/**` define the regression contract for this module. They must continue to pass before any mutation is merged. Per `.stokd/meta/packages/sui-editor/SC_TEST.md`, the plugin tests currently require a `describeEditor` harness at `test/utils/editor-view/describeEditor.tsx` that does not yet exist; the correct response to a failing plugin test is to build that harness, not to delete the test or mark it `.skip`.

### Applies To
- `packages/sui-editor/test/**`
- `packages/sui-editor/src/**/*.test.tsx`
- `packages/sui-editor/src/**/*.test.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/editor typescript`
- `pnpm test:unit -- packages/sui-editor` (from the repo root; falls back to `pnpm test` if the project script is renamed)
- manual: if a plugin test is failing because `describeEditor` is missing, add the harness or open a governed task to land it — do not `.skip` or delete the test.

## AX-MOD-SUI-EDITOR-006: `sui-editor` is a library; runtime entry points are React, not HTTP
This package exports a React component (`Editor`), a context provider (`EditorProvider`), and a runtime class (`EditorEngine`). It must not introduce server-side HTTP routes, Next.js API handlers, NestJS controllers, or CLI binaries; per the workspace's media-API boundary rule (see `.stokd/meta/SC_CONTEXT.md`), non-media business routes belong in `docs/pages/api/**` and media-component endpoints belong in `packages/sui-media-api`. Adding `bin`, `pages/api`, or an Express/Nest entry to this package is out of scope.

### Applies To
- `packages/sui-editor/package.json`
- `packages/sui-editor/src/**`

### Acceptance Checks
- manual: confirm `packages/sui-editor/package.json` has no `bin` field and no `main` other than `src/index.ts` (build output is `build/`).
- manual: confirm there is no `pages/api` directory and no `Controller` decorator imports anywhere under `packages/sui-editor/src/**`.

## AX-MOD-SUI-EDITOR-007: `StokedUiEditorApp` singleton + its `.sue`/`.suvid`/`.sua` MIME registration are the editor file contract
`StokedUiEditorApp` (`src/Editor/StokedUiEditorApp.ts`) is a private-constructor singleton (`getInstance()`) that registers exactly three MIME types via `SUIMime.make`: `.sue` (`editor-project`, the `defaultInputFileType`), `.suvid` (`editor-video`, the `defaultOutputFileType`/`videoFileType`), and `.sua` (`editor-audio`, `audioFileType`), wired to `AppFileFactory`/`AppOutputFileFactory`. These extensions, MIME ids, and the `defaultInputFileType` are a cross-package contract: `EditorProvider` resolves the IndexedDB store with `getDbProps(state.app.defaultInputFileType, props.localDb)`, and saved/opened projects use the `.sue` extension. Changing an extension, a MIME id, the singleton shape, or which type is `defaultInputFileType` is a breaking change that desyncs file open/save and the persistence store wiring, and requires a coordinated migration.

### Applies To
- `packages/sui-editor/src/Editor/StokedUiEditorApp.ts`
- `packages/sui-editor/src/EditorProvider/EditorProvider.tsx`

### Acceptance Checks
- `pnpm --filter @stoked-ui/editor typescript`
- `grep -nE "\.sue|\.suvid|\.sua|defaultInputFileType|getInstance" packages/sui-editor/src/Editor/StokedUiEditorApp.ts` shows the three extensions and the singleton accessor unchanged.
- manual: confirm `EditorProvider` still passes `state.app.defaultInputFileType` into `getDbProps(...)` so the IDB store keeps resolving from the `.sue` input type.
