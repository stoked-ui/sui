# AXIOMS

## AX-MOD-FILE-EXPLORER-001: Public barrel defines the contract

The set of symbols re-exported by `src/index.ts` (components, hooks, models,
plugins, icons, theme augmentation, feature flags) is the public contract of
`@stoked-ui/file-explorer`. Removing or renaming any export is a breaking
change that requires a governed task with explicit consumer review (editor,
docs demos, showcase cards).

### Applies To
- `packages/sui-file-explorer/src/index.ts`
- `packages/sui-file-explorer/src/internals/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/file-explorer typescript`
- manual: confirm every removed/renamed symbol has been searched repo-wide (`docs/`, `packages/sui-editor`) and updated.

## AX-MOD-FILE-EXPLORER-002: Plugin order in FILE_EXPLORER_PLUGINS is load-bearing

The tuple `FILE_EXPLORER_PLUGINS` in `src/FileExplorer/FileExplorer.plugins.ts`
declares plugin execution order: `useFileExplorerFiles` →
`useFileExplorerExpansion` → `useFileExplorerSelection` →
`useFileExplorerFocus` → `useFileExplorerKeyboardNavigation` →
`useFileExplorerIcons` → `useFileExplorerGrid` → `useFileExplorerDnd`.
Later plugins read state set by earlier ones. Reordering or inserting plugins
in the middle of this list MUST be accompanied by a full plugin-test run.

### Applies To
- `packages/sui-file-explorer/src/FileExplorer/FileExplorer.plugins.ts`
- `packages/sui-file-explorer/src/internals/plugins/**`

### Acceptance Checks
- `pnpm --filter @stoked-ui/file-explorer typescript`
- manual: run the `useFileExplorer*` plugin test suites under the root mocha command (note: see SC_TEST.md §1.2 — the harness import paths currently need fixing before tests run cleanly).

## AX-MOD-FILE-EXPLORER-003: File validation is the security gate for external drops

`src/internals/plugins/useFileExplorerDnd/fileValidation.ts`
(`isDangerousExtension`, `isAllowedMimeType`, `isAcceptableFileSize`,
`validateFile`, `validateFiles`, `getRejectionReason`) is the only sanctioned
boundary between OS-originated drops and consumer code. Loosening the default
denylist/allowlist, raising default size caps, or bypassing `validateFiles`
on the external-drop path requires a security review.

### Applies To
- `packages/sui-file-explorer/src/internals/plugins/useFileExplorerDnd/fileValidation.ts`
- `packages/sui-file-explorer/src/internals/plugins/useFileExplorerDnd/useFileExplorerDnd.tsx`
- `packages/sui-file-explorer/src/FileDropzone/**`

### Acceptance Checks
- manual: confirm every external-drop code path (`monitorForExternal`, `dropTargetForExternal`, `FileDropzone` `onDrop`) routes incoming `File` objects through `validateFile`/`validateFiles` before invoking consumer callbacks.
- manual: audit any added entry to allowed MIME types or extension lists for executable/script formats.

## AX-MOD-FILE-EXPLORER-004: FileBase shape is a cross-package data contract

`FileBase` / `FileBaseInput` / `FileId` in `src/models/items.ts` are consumed
by `@stoked-ui/editor`, the docs demos under `docs/data/file-explorer/**`,
and showcase cards. Required-field additions, type-narrowing changes, or
`mediaType` enum bumps (which originate in `@stoked-ui/media`) ripple
repo-wide; such changes require a coordinated migration.

### Applies To
- `packages/sui-file-explorer/src/models/items.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/file-explorer typescript`
- manual: run `pnpm typescript` at the monorepo root to catch consumer breakage in `packages/sui-editor` and `docs/`.

## AX-MOD-FILE-EXPLORER-005: Theme augmentation must mirror new public slots and classes

Adding a `MuiFileExplorer*`, `MuiFile`, or `MuiFileExplorerTabs` slot or
classKey requires a matching entry in `src/themeAugmentation/`. Without it,
consumer themes lose type safety on the new surface.

### Applies To
- `packages/sui-file-explorer/src/themeAugmentation/**`
- `packages/sui-file-explorer/src/**/fileExplorer*Classes.ts`
- `packages/sui-file-explorer/src/**/fileClasses.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/file-explorer typescript`
- manual: verify `themeAugmentation.spec.ts` covers the new slot/class.

## AX-MOD-FILE-EXPLORER-006: Existing tests must keep passing before mutation

Tests under `packages/sui-file-explorer/src/**/*.test.{ts,tsx}` and
`themeAugmentation.spec.ts` define the observable behavior contract of this
module. They must continue to pass (with the harness-path fix from
SC_TEST.md §1.2 applied) before any mutation to this module is merged.

### Applies To
- `packages/sui-file-explorer/src/**/*.test.{ts,tsx}`
- `packages/sui-file-explorer/src/themeAugmentation/themeAugmentation.spec.ts`

### Acceptance Checks
- `pnpm test -- 'packages/sui-file-explorer/src/**/*.test.{ts,tsx}'`
- manual: if a test is intentionally changed, update SC_TEST.md to reflect the new expectation.
