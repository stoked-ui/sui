# @stoked-ui/stokd axioms

Directory-scoped invariants for the Stokd "Current Activity" UX library.

## AX-MOD-SUISTOKD-VIEWMODEL-SOURCE-OF-TRUTH: @stoked-ui/stokd Owns The UX View-Model Types

`@stoked-ui/stokd` is the single source of truth for the Current Activity UX
view-model types (defined in `src/types`). It MUST NOT import any consumer-app
package (no `@stokd-cloud/*`, no host SDK, no editor API). Consumers — the Stokd
web dashboard, the VS Code extension webview, and any future host — map their own
domain data into these types via a thin adapter, and the platform's
`@stokd-cloud/shared` re-exports these types rather than the reverse.

### Why

The library is reused across multiple hosts that each own different domain
models. A one-way type flow (SUI → consumers) keeps the library host-agnostic and
prevents a circular cross-repo dependency.

### How to apply

- Define view-model interfaces in `packages/sui-stokd/src/types` and export them
  from `src/index.ts`; consumers import them and write adapters.
- Never add `@stokd-cloud/*` (or any host package) to `sui-stokd` dependencies.

### Acceptance Checks

- `! grep -rn "@stokd-cloud/" packages/sui-stokd/src`

### Enforcement

computational

## AX-MOD-SUISTOKD-CSS-VAR-THEMED: Components Are Themed Exclusively Via CSS Custom Properties

`@stoked-ui/stokd` components are styled exclusively via CSS custom properties
(`var(--sui-*)`, and `color-mix()` of them for tints). They use no Tailwind, no
required MUI `ThemeProvider`, and no hard-coded hex/rgb theme colors in component
source. Concrete color literals live only in the default theme token CSS file
(`src/theme/tokens.css`), which a host overrides — the web maps the tokens to its
`github-*`/`accent-*` Tailwind tokens, the VS Code webview maps them to
`--vscode-*` theme tokens.

### Why

The same components must render under web Tailwind tokens and under VS Code theme
tokens; a hard-coded color or a Tailwind dependency would break one host.

### How to apply

- Reference colors via `var(--sui-*)`; ship the default theme stylesheet; hosts
  override the variables.
- Keep concrete color literals out of component `.tsx`/`.ts` source.

### Acceptance Checks

- `! grep -rnE "tailwind" packages/sui-stokd/src`
- `! grep -rnE "#[0-9a-fA-F]{3,6}|rgb\\(" $(find packages/sui-stokd/src -name '*.tsx')`

### Enforcement

computational

## AX-MOD-SUISTOKD-PRESENTATIONAL-NO-IO: Components Are Purely Presentational

`@stoked-ui/stokd` components are purely presentational — no data fetching,
routing, or network/auth side effects. All data enters via props (the view-model)
and all actions exit via callbacks (`onCancel`, `onContinue`, `onOpenTask`, …).

### Why

Presentational purity lets every host own its own data/realtime/routing and reuse
the components unchanged.

### How to apply

- Components take a view-model + callback props.
- No `fetch`/`axios`/`apiClient`/`react-router` imports inside the library.

### Acceptance Checks

- `! grep -rnE "fetch\\(|axios|apiClient|react-router" packages/sui-stokd/src`

### Enforcement

computational
