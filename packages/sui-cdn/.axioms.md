# AXIOMS

## AX-MOD-CDN-001: `src/index.ts` Is The Sole Public Contract
The public surface of `@stoked-ui/cdn` is exposed exclusively through the
`src/index.ts` barrel (`CdnBrowser`, `createCdnApi`, `collectDroppedEntries`,
`beginDesktopDownload`, `getFileKind`, `formatBytes`, `formatTimestamp`,
`buildCrumbs`, `normalizePrefix`, `mockObjects`, and the exported types);
consumers MUST import from `@stoked-ui/cdn` (never deep paths), and adding,
removing, or renaming a barrel export is a contract change requiring a governed
task with downstream consumer review (`packages-internal/cdn-sui`, docs admin).

### Applies To
- `packages/sui-cdn/src/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/cdn typescript`
- `grep -rnE "from ['\"]@stoked-ui/cdn/(src|CdnBrowser|CdnApi|utils|data)" packages packages-internal docs` returns no deep cross-package imports.

## AX-MOD-CDN-002: All CDN Network Access Goes Through `createCdnApi` With Credentials
Every CDN HTTP request issued by this package MUST be made via `createCdnApi`
against a single configurable base (default `/api/cdn`) with the trailing slash
stripped, MUST send `credentials: 'include'`, and MUST use the established
endpoint shapes (`/folders`, `/delete`, `/move`, `/permissions`, `/export`,
`/contents`, `/upload/initiate`, `/upload/{id}/status`, `/upload/{id}/urls`,
`/upload/{id}/part/{n}`, `/upload/{id}/complete`); these paths/bodies are a
coordinated contract with `docs/pages/api/cdn/**` and MUST NOT diverge
unilaterally.

### Applies To
- `packages/sui-cdn/src/CdnApi/CdnApi.ts`
- `packages/sui-cdn/src/utils/contents.ts` (`fetchRemote`)

### Acceptance Checks
- `grep -nE "credentials: 'include'" packages/sui-cdn/src/CdnApi/CdnApi.ts packages/sui-cdn/src/utils/contents.ts` shows every fetch path opts in.
- manual: any new endpoint path added to `createCdnApi` has a matching handler under `docs/pages/api/cdn/**`.
- `pnpm --filter @stoked-ui/cdn typescript`

## AX-MOD-CDN-003: Resumable Multipart Upload Invariants Are Preserved
`createCdnApi().uploadFile` MUST remain resumable and fault-tolerant:
`uploadChunk` retries non-abort failures up to `MAX_RETRIES` (3) with backoff,
MUST NOT retry on `AbortError`, and MUST throw when a chunk response lacks an
`ETag`; presigned URLs are fetched in batches of `URL_BATCH_SIZE` (50); the
upload session id is fingerprinted into `localStorage` (`cdn-upload:{path}:{name}:{size}:{lastModified}`),
reused on resume via the `status` endpoint, cleared when the stored session is
invalid, and cleared on successful `complete`.

### Applies To
- `packages/sui-cdn/src/CdnApi/CdnApi.ts` (`uploadFile`, `uploadChunk`, `uploadFingerprint`, `getStoredSessionId`/`setStoredSessionId`/`clearStoredSessionId`)

### Acceptance Checks
- `grep -nE "MAX_RETRIES|URL_BATCH_SIZE|AbortError|did not include an ETag" packages/sui-cdn/src/CdnApi/CdnApi.ts` shows the guards intact.
- manual: changes to the multipart flow re-verify resume, abort, monotonic `onProgress` ending at 100, and fingerprint cleanup (see SC_TEST.md §6 Tier 3).

## AX-MOD-CDN-004: Browser Storage And Platform Access Must Be Failure-Safe And SSR-Guarded
All `localStorage` access (upload session ids and per-prefix view mode
`cdn-view-mode-{prefix}`) MUST be wrapped in try/catch that swallows failures
(private mode / quota), and any code path that reads `window` outside React
effects MUST guard with `typeof window !== 'undefined'`; the package is
browser-targeted and MUST NOT crash during SSR or when storage is unavailable.

### Applies To
- `packages/sui-cdn/src/CdnApi/CdnApi.ts` (`getStoredSessionId`, `setStoredSessionId`, `clearStoredSessionId`)
- `packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` (`readStoredViewMode`, `writeStoredViewMode`, `currentUrl`)
- `packages/sui-cdn/src/utils/contents.ts` (`getContents` localhost fallback)

### Acceptance Checks
- `grep -nE "try \{|typeof window" packages/sui-cdn/src/CdnApi/CdnApi.ts packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx packages/sui-cdn/src/utils/contents.ts` shows storage/window access is guarded.
- `pnpm -w build` (Babel pipeline) green — no SSR-incompatible top-level browser access.

## AX-MOD-CDN-005: Management Actions Are Admin-Gated; Public View Is Read-Only
Mutating UI affordances (create folder, rename, move/drag, delete, permissions
editor, restrict) MUST only render and fire when `canManage` is true
(`auth.status === 'authenticated' && auth.user?.role === 'admin'`); when no
`authEndpoint` is supplied the auth chrome and all management controls MUST be
hidden, and credential-expiry / 401 conditions MUST force a logout redirect via
`shouldForceLogout`.

### Applies To
- `packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` (`canManage`, `shouldForceLogout`, `isCredentialFailure`, `useAuthSession`)

### Acceptance Checks
- `grep -nE "canManage|shouldForceLogout|role === 'admin'" packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` shows the gate is applied to management controls.
- manual: render `CdnBrowser` without `authEndpoint` → no auth UI and no create/rename/move/delete/permissions controls appear.

## AX-MOD-CDN-006: Pure Utility Semantics Are Observable Behavior
The pure helpers in `utils/contents.ts` define observable behavior that the
component and downstream consumers depend on and MUST NOT silently change:
`normalizePrefix` collapses slashes / strips a leading slash / appends a
trailing slash; `getFileKind` maps extensions to `video|image|download|code|file`
(case-insensitive); `formatBytes(0) === '0 B'`; `formatTimestamp('') === 'Unknown'`;
`fromFlatObjects` groups flat keys into sorted folders/objects within the prefix;
`parseS3Xml` throws `'Invalid XML response from contents endpoint'` on a parser
error. Reword/removal requires updating any docs/tests that reference these.

### Applies To
- `packages/sui-cdn/src/utils/contents.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/cdn typescript`
- manual: behavior changes to these helpers update SC_TEST.md §1/§6 expectations and any consuming view logic (e.g. `isMediaHeavy` gallery auto-switch).

## AX-MOD-CDN-007: Styling Is A Side-Effect Stylesheet Scoped Under `.suiCdnBrowser`
All component styling ships as a single side-effect stylesheet
(`src/styles.css`, declared via `package.json` `sideEffects: ["*.css"]`) that host
apps import as `@stoked-ui/cdn/styles.css`; the component source MUST NOT
`import './styles.css'` itself, every rule MUST stay scoped under the
`.suiCdnBrowser` root class (and the documented sub-element class names the
`.tsx` emits), and the root element MUST keep rendering `className="suiCdnBrowser"`.
Renaming the root class or the public stylesheet path is a consumer-facing
contract change (it silently breaks host styling and the `styles.css` import).

### Applies To
- `packages/sui-cdn/src/styles.css`
- `packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` (root `className="suiCdnBrowser"`)
- `packages/sui-cdn/package.json` (`sideEffects`)

### Acceptance Checks
- `grep -n "suiCdnBrowser" packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx packages/sui-cdn/src/styles.css` shows the root class on the element and as the CSS scope.
- `grep -rn "styles.css" packages/sui-cdn/src` returns no in-source import (host-consumed only).
- `grep -n '"sideEffects"' packages/sui-cdn/package.json` shows `["*.css"]` so the stylesheet survives tree-shaking.

## AX-MOD-CDN-008: `hideRootFiles` Defaults To `true` To Protect The Bucket Root
`CdnBrowser` MUST default `hideRootFiles` to `true`, and when the active prefix is
the bucket root it MUST hide root-level objects from the listing (the S3 bucket
root holds SPA shell files that must not be exposed); folders are never hidden.
Flipping this default, or removing the root-prefix guard, is a behavior/security
change requiring a governed task.

### Applies To
- `packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` (`hideRootFiles` default, `isRoot`/`visible`)
- `packages/sui-cdn/src/CdnBrowser/CdnBrowser.types.ts` (`hideRootFiles?` `@default true`)

### Acceptance Checks
- `grep -nE "hideRootFiles = true|isRoot|visible = \(hideRootFiles" packages/sui-cdn/src/CdnBrowser/CdnBrowser.tsx` shows the default and the root-prefix gate.
- manual: render `CdnBrowser` at the root prefix without overriding the prop → root-level objects are hidden, folders still render.
