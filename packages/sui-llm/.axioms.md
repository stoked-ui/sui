# @stoked-ui/llm axioms

Directory-scoped invariants for the Stoked UI LLM agent UI library.

## AX-MOD-SUILLM-PRESENTATIONAL-NO-IO: Components Are Purely Presentational

`@stoked-ui/llm` components are purely presentational — no data fetching, routing, or network/auth side effects. All data enters via props and all actions exit via callbacks (`onSelectAgent`, `onClose`, `onSend`, …). This includes no `setInterval`/`localStorage` for goal simulation, no `fetch`/`axios` for API calls, and no `next/*` imports.

### Why

Presentational purity lets every host (HQ, CLI tools, dashboards) own its own data/realtime/routing and reuse the components unchanged.

### How to apply

- Components take a view-model + callback props.
- No `fetch`/`axios`/`apiClient`/`react-router`/`next/*` imports inside the library.
- No `setInterval` or `localStorage` hooks for simulated state — all data flows in via props.

### Enforcement

## AX-MOD-SUILLM-MUI-V5-THEMED: Components Use MUI v5 + Emotion, Not CSS Variables

`@stoked-ui/llm` components are styled using Material-UI v5 (`@mui/material` 5.17.1) with Emotion runtime styling (`@emotion/react`/`@emotion/styled`). They do NOT use CSS custom properties (`var(--sui-*)`) for theming (unlike `@stoked-ui/stokd`), and they do NOT use Tailwind or CSS‑in‑JS tokens.

### Why

The LLM agent UI needs dense, rich interactive widgets (tabs, selectors, progress bars, cards) that MUI v5 provides out‑of‑the‑box, with a coherent design system and accessible defaults. Emotion gives us runtime‑dynamic styling (agent colors, status indicators) without needing a CSS‑variable mapping layer.

### How to apply

- Import from `@mui/material`, `@mui/icons‑material`.
- Style via `sx` prop, `styled()` from `@emotion/styled`, or MUI’s `makeStyles`/`useTheme` hooks.
- Never add `tailwindcss` or equivalent utility‑class dependencies.
- Keep color literals in MUI theme objects or Emotion style blocks, not in CSS variable overrides.

### Acceptance Checks

- `! grep -rnE "tailwind" packages/sui-llm/src`
- `! grep -rn "from.*@stoked-ui/stokd" packages/sui-llm/src` (no CSS‑var leakage)

### Enforcement

computational

## AX-MOD-SUILLM-BARREL-CONTRACT: `src/index.ts` Is The Sole Public API Surface

`@stoked-ui/llm` exports all its public symbols through the barrel file `src/index.ts`. Consumers must import from the package root (`@stoked-ui/llm`), never from internal paths (`@stoked-ui/llm/src/…` or `@stoked-ui/llm/internals/…`).

### Why

A single entry point guarantees consumers see a stable, versioned API; internal refactoring (file moves, re‑exports) doesn't break them. It also lets us tree‑shake unused exports at build time.

### How to apply

- Every public type, component, and helper must be re‑exported from `src/index.ts`.
- Never export directly from a component's own file — export from its `index.ts` and re‑export that.
- Test files may import from `src/…` directly (they are internal).

### Acceptance Checks

- `! grep -rnE "from\\s+['\"]@stoked-ui/llm/(src|internals)/" packages docs` (in the whole monorepo)
- `src/index.ts` exports all public symbols documented in README.md.

### Enforcement

computational
