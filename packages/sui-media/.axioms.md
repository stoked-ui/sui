# AXIOMS

## AX-MOD-SUIMEDIA-001: Public surface is the index barrel; deep imports are not contract
The package's public API is exactly what `src/index.ts` exports plus `src/server/index.ts` (`createMediaHandler`). Consumers must import from `@stoked-ui/media` or `@stoked-ui/media/server`. Deep imports into `src/MediaFile/`, `src/api/`, `src/hooks/`, `src/components/`, etc. are not supported. Adding, removing, or renaming a top-level export is a breaking change that requires a governed task.

### Applies To
- `packages/sui-media/src/index.ts`
- `packages/sui-media/src/server/index.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/media build` succeeds (modern + node + stable + types).
- `pnpm --filter @stoked-ui/editor typescript && pnpm --filter @stoked-ui/timeline typescript && pnpm --filter @stoked-ui/file-explorer typescript` all pass — these workspace consumers must continue to resolve every name they import.
- manual: any rename/removal in `src/index.ts` is mirrored across `docs/src/products.tsx`, `docs/data/media/docs/**`, and the consuming packages above.

## AX-MOD-SUIMEDIA-002: `IMediaFile`, `MediaType`, and `Command` are cross-package wire types
`IMediaFile` field shape (`id`, `mediaType`, `path`, `url`, `media: Settings`, `children: IMediaFile[]`, `created`, `getUrl()`), the `MediaType` union (`'doc' | 'file' | 'folder' | 'image' | 'pdf' | 'trash' | 'video' | 'lottie' | 'audio' | string` — the string fallback is intentional, do not narrow), and the `Command` undo/redo interface (`execute()`/`undo()`) are shared contracts with `@stoked-ui/editor`, `@stoked-ui/timeline`, and `@stoked-ui/file-explorer`. `getMediaType()` must continue to return `'file'` for unknown mimes.

### Applies To
- `packages/sui-media/src/MediaFile/IMediaFile.ts`
- `packages/sui-media/src/MediaFile/MediaFile.ts`
- `packages/sui-media/src/MediaType/MediaType.ts`
- `packages/sui-media/src/WebFile/WebFile.ts` (Command interface)

### Acceptance Checks
- `pnpm --filter @stoked-ui/media test` passes (the `MediaCard`, abstractions, and integration-backward-compatibility suites guard the surface).
- `pnpm --filter @stoked-ui/editor typescript && pnpm --filter @stoked-ui/timeline typescript && pnpm --filter @stoked-ui/file-explorer typescript` — type-cascade guard for downstream consumers.
- manual: confirm `getMediaType(undefined)`, `getMediaType('')`, and `getMediaType('application/octet-stream')` all still return `'file'`.

## AX-MOD-SUIMEDIA-003: Documented runtime quirks must not silently regress
Three known fixes are load-bearing for downstream UI and must not be reverted without a governed migration:
1. `createSettings` Proxy on `MediaFile.media` — properties set via `Object.assign` do not always propagate through React state; the editor's `DetailView` falls back to the live DOM `<video>` element for duration/width/height. Replacing the proxy or "fixing" propagation requires auditing every `Object.assign(file.media, ...)` consumer in `@stoked-ui/editor` first.
2. `ScreenshotStore` empty-store render path — rendering must work when `count === 0` (the editor previously blocked thumbnail generation on empty stores).
3. `WebFile` IndexedDB save — the version-entry creation guard handles the case where the IDB version entry is missing.

### Applies To
- `packages/sui-media/src/MediaFile/MediaFile.ts`
- `packages/sui-media/src/MediaFile/Metadata/ScreenshotStore.ts`
- `packages/sui-media/src/MediaFile/Metadata/ScreenshotQueue.ts`
- `packages/sui-media/src/WebFile/WebFile.ts`

### Acceptance Checks
- `pnpm --filter @stoked-ui/media test` passes.
- manual: open the docs site at port 5199 (`pnpm docs:dev`), open the editor (`/x/editor`), drop a video clip, scrub the timeline — confirm thumbnails render and DetailView duration/width/height populate.
- manual: trigger a project save in the editor against a previously-untouched IDB origin — confirm no "version entry missing" exception.

## AX-MOD-SUIMEDIA-004: Server-only code stays behind the `/server` subpath
`createMediaHandler` and any other code that imports `next/server` (`NextRequest`/`NextResponse`) or relies on `process.env.STOKED_MEDIA_API_KEY` must only be reachable from `@stoked-ui/media/server`. The browser entry (`src/index.ts`) must never transitively import `next/server` or read `STOKED_MEDIA_API_KEY`. The private API key must never leak into client bundles.

### Applies To
- `packages/sui-media/src/server/**`
- `packages/sui-media/src/index.ts` (must not import from `src/server/`)

### Acceptance Checks
- `pnpm --filter @stoked-ui/media build` succeeds and the modern/stable/node browser bundles contain no reference to `next/server` or `STOKED_MEDIA_API_KEY`.
- manual: `rg -n "from 'next/server'|from \"next/server\"" packages/sui-media/src` returns matches only under `src/server/`.
- manual: confirm `createMediaHandler({ privateKey, sourceApiUrl?, localPath? })` still returns `{ GET, POST, PUT, DELETE }` per `API_CLIENT_README.md`.

## AX-MOD-SUIMEDIA-005: Upload + API client contracts mirror `sui-common` / `sui-media-api`
`MediaApiClient` and `UploadClient` paths, the upload chunk bounds (5 MiB ≤ chunkSize ≤ 100 MiB, 1–50 partNumbers), the `^(video|image)/[a-z0-9.+-]+$` mime regex on `InitiateUploadDto`, and the DTO/response types re-exported via `src/api/types.ts` from `@stoked-ui/common` are the wire contract with `@stoked-ui/media-api`. Loosening or restructuring requires a coordinated change in `sui-common`, `sui-common-api`, and `sui-media-api`.

- `packages/sui-media/src/api/types.ts`

NB: there is **no** `src/api/__tests__/` suite yet (verified 2026-06-06) — the `MediaApiClient`/`UploadClient` wire contract is currently guarded by the TypeScript build, the `sui-media-api` typecheck, and manual e2e, not by unit tests.

## AX-MOD-SUIMEDIA-006: TanStack Query cache keys are a public invalidation contract
The hook cache key shapes (verified against source) — `['media', id]` (`useMediaItem`; also the delete-removal key), `['media', 'list', params]` (`useMediaList`, invalidated by the mutation hooks via the `['media', 'list']` prefix), `['media-gallery', source]` (`useMediaGallery`), `['uploads', 'active']` (`useActiveUploads`), and the `mediaQueryKeys` factory rooted at `['media']` with `metadata`/`thumbnails`/`search`/`extraction` sub-trees (`useMediaMetadataCache`) — are part of the public contract. Host apps invalidate by these prefixes. Reshaping any of them is a breaking change and requires a governed task plus consumer notification.

### Applies To
- `packages/sui-media/src/hooks/**`
- `packages/sui-media/src/hooks/MediaApiProvider.tsx`

### Acceptance Checks
- manual: `rg -n "queryKey|invalidateQueries" packages/sui-media/src/hooks` confirms the keys at source — `['media', id]` (`useMediaItem.ts:48`, removal at `useMediaDelete.ts:63`), `['media', 'list', params]` (`useMediaList.ts:68`; mutations invalidate the `['media', 'list']` prefix in `useMediaUpload.ts:178`/`useMediaUpdate.ts:63`/`useMediaDelete.ts:67`), `['media-gallery', source]` (`useMediaGallery.ts:16`), `['uploads', 'active']` (`useResumeUpload.ts:40`), plus the `mediaQueryKeys` factory (`useMediaMetadataCache.ts`).

## AX-MOD-SUIMEDIA-008: MediaClass branded playback follows a fixed phase order and stays host-URL-agnostic
`useMediaClassPlayback` is the state machine for MediaClass branded content. Its `PlaybackPhase` advances strictly `beforeIdent → main → afterIdent → complete`; `getInitialPhase` MUST return `'main'` when disabled or when no `mediaClass` is supplied, and start at `'beforeIdent'` only when `mediaClass.beforeIdent` is configured. The hook MUST remain framework-agnostic — it resolves every URL through the injected `getMediaUrl(type, id)` and MUST NOT import Next.js or any host-app URL helper. The `MediaClassConfig` / `IdentRef` / `VideoBugConfig` / `PlaybackPhase` shapes are the config contract for branded playback. `onPlaybackComplete` fires exactly once per terminal transition into `'complete'`.

### Applies To
- `packages/sui-media/src/components/MediaViewer/hooks/useMediaClassPlayback.ts`
- `packages/sui-media/src/components/MediaViewer/MediaViewer.types.ts` (`PlaybackPhase`, `IdentRef`, `VideoBugConfig`, `MediaClassConfig`)

### Acceptance Checks
- `pnpm --filter @stoked-ui/media test` passes the `describe('useMediaClassPlayback')` suite in `src/components/MediaViewer/__tests__/hooks.test.ts`.
- manual: `rg -n "next/|from 'next" packages/sui-media/src/components/MediaViewer/hooks/useMediaClassPlayback.ts` returns no matches (no Next.js coupling).
- manual: confirm `useMediaClassPlayback({ mediaClass: null, ... })` reports `phase === 'main'` and `currentVideoUrl === getMediaUrl('videos', mainVideoId)`.

## AX-MOD-SUIMEDIA-009: The root barrel is the only public surface; sub-barrel and lazy helpers are not contract
`package.json` declares no `exports` map (`main: src/index.ts`, `publishConfig.directory: build`), so subpaths resolve via the mirrored `build/` tree. The supported public surfaces are exactly `@stoked-ui/media` (root barrel) and `@stoked-ui/media/server` (`createMediaHandler`). Anything reachable only through a deep path — `useMediaClassPlayback` and the MediaClass types via the MediaViewer hooks sub-barrel, the `src/components/lazy.tsx` helpers, and the internal `useMediaMetadataCache` / `useHybridMetadata` / `useServerThumbnail` hooks — is NOT part of the contract and MUST NOT be relied on by external consumers. Adding an `exports` map is a contract change that MUST keep both `.` and `./server` reachable; promoting a deep path to public (e.g. wiring up `@stoked-ui/media/lazy`) requires a governed task.

### Applies To
- `packages/sui-media/package.json` (`main`, `exports`, `publishConfig`)
- `packages/sui-media/src/index.ts`
- `packages/sui-media/src/server/index.ts`

### Acceptance Checks
- `node -e "const p=require('./packages/sui-media/package.json'); if (p.exports && !(p.exports['.'] && p.exports['./server'])) { throw new Error('exports map must keep . and ./server'); }"` exits 0.
- manual: `rg -n "@stoked-ui/media/(?!server)" docs packages --glob '!**/build/**'` surfaces no external deep-path imports beyond `@stoked-ui/media` and `@stoked-ui/media/server`.
